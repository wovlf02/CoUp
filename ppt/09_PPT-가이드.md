## 슬라이드 5: 해결 방안 (CoUp)

## 슬라이드 11: 시스템 아키텍처

**내용:**
```
Client (Browser)
      ↓ HTTPS              ↓ WebSocket
┌─────────────┐      ┌─────────────┐
│  Next.js    │      │  Signaling  │
│    App      │      │   Server    │
└──────┬──────┘      └──────┬──────┘
       │                    │
       ▼                    ▼
  PostgreSQL            Redis (Pub/Sub)
```

**핵심 포인트:**
- 분리형 아키텍처: HTTP API와 WebSocket 서버 독립
- Redis Adapter로 다중 서버 지원
- Docker Compose로 서비스 오케스트레이션

**내용:**

**시각 자료:**
```
User ──1:N── Study (소유)
User ──N:M── Study (멤버) via StudyMember
Study ──1:N── Message
Study ──1:N── Notice
Study ──1:N── Event
```

## 슬라이드 13: 핵심 코드 - API

**내용:**
- Next.js API Routes 예시
- TanStack Query로 데이터 페칭
- Prisma로 DB 쿼리

**코드 (간략화):**
```javascript
// API Route
export async function GET(request) {
  const session = await requireAuth()
  const studies = await prisma.study.findMany({
    where: { isPublic: true },
    include: { _count: { select: { members: true } } }
  })
  return NextResponse.json({ data: studies })
}
```
