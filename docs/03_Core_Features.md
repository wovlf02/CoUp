# 03. 핵심 기능 설계

- **설계 원칙**
  - **All-in-One**: 사용자가 다른 툴을 사용할 필요가 없도록, 스터디의 전 과정을 플랫폼 내에서 완결시킨다.
  - **Simplicity**: 강력한 기능을 제공하되, 사용자가 직관적으로 이해하고 사용할 수 있도록 UI/UX를 설계한다.
  - **Data-Driven**: 데이터를 기반으로 사용자에게 유의미한 피드백과 동기부여를 제공한다.

- **MVP 개발 전략**
  - MVP 단계의 모든 기능은 **Next.js 풀스택 방식**으로 개발됩니다. 즉, 특정 기능에 대한 프론트엔드 UI와 백엔드 API 로직(in API Routes)을 하나의 작업 단위로 묶어, Mock API 없이 실제 데이터베이스와 연동하여 직접 구현합니다.

## 1. MVP(Minimum Viable Product) 기능 목록

MVP는 "**흩어져 있는 스터디를 모으고, 기본적인 활동을 가능하게 한다**"는 핵심 가치에 집중합니다. 최소한의 기능으로 시장의 반응을 빠르게 테스트하고, 사용자 피드백을 통해 서비스를 개선하는 것을 목표로 합니다.

### 1.1. 사용자 및 인증 (User & Auth)

- [ ] **소셜 로그인**: GitHub, Google, Kakao 계정을 이용한 간편 회원가입 및 로그인 (NextAuth.js 사용)
- [ ] **기본 프로필 관리**: 닉네임, 프로필 사진, 자기소개 조회 및 수정

### 1.2. 스터디 그룹 (Study Group)

- [ ] **그룹 생성**: 스터디명, 소개, 카테고리, 모집 인원, 공개/비공개 설정
- [ ] **그룹 탐색 및 검색**: 카테고리별, 키워드별 스터디 그룹 검색 기능
- [ ] **그룹 가입 및 탈퇴**: 가입 신청, 승인/거절, 탈퇴 기능
- [ ] **그룹원 관리**: 그룹장(Owner), 멤버(Member) 권한 관리, 멤버 초대 및 강퇴

### 1.3. 스터디 활동 (Study Activities)

- [ ] **그룹 전용 채팅**: 실시간 소통을 위한 그룹 채팅 채널 (서드파티 서비스 연동)
- [ ] **통합 화상 스터디**: WebRTC 기반의 다자간 화상 스터디 기능 (화면 공유, 캠 온/오프, 음소거, 다른 참여자 마이크 음소거)
- [ ] **공지사항**: 그룹장이 중요 내용을 전달하는 공지 기능
- [ ] **파일 공유**: 간단한 학습 자료(이미지, PDF 등) 업로드 및 다운로드 (AWS S3 연동)
- [ ] **그룹 캘린더**: 스터디 일정, 모임, 과제 마감일 등을 공유하는 캘린더

### 1.4. 기본 알림 (Notification)

- [ ] **웹 기반 알림**: 새로운 채팅 메시지, 가입 승인, 공지 등록 시 실시간 알림 (서드파티 서비스 연동)

## 2. 정식 버전 (V1) 및 향후 확장 기능 (Post-MVP)

MVP 출시 및 소프트웨어 저작권 등록 이후, 사용자 피드백과 데이터를 바탕으로 플랫폼의 경쟁력을 강화하고 사용자 경험을 고도화하는 기능들을 순차적으로 개발합니다. 이 단계에서는 백엔드를 Spring Boot 등 별도의 전문 프레임워크로 전환하는 것을 고려할 수 있습니다.

### 2.1. 'All-in-One' 경험 강화

- [ ] **칸반 보드**: 그룹 과제, 프로젝트 진행 상황을 시각적으로 관리하는 칸반 보드
- [ ] **공유 문서**: 그룹원들이 실시간으로 함께 문서를 편집하는 기능

### 2.2. 지능형 매칭 및 관리 자동화

- [ ] **지능형 그룹 추천**: 사용자 프로필, 활동 이력, 관심사를 분석하여 최적의 스터디 그룹을 추천하는 AI 알고리즘
- [ ] **'그룹 마스터' 시스템 (관리 자동화)**
  - [ ] **자동 출석 체크**: 화상 스터디 참여 시 자동 출결 확인
  - [ ] **예치금 및 벌금 시스템**: 그룹 규칙에 따른 예치금 설정 및 벌금 자동 정산/분배

### 2.3. 데이터 기반 동기 부여

- [ ] **개인/그룹 대시보드**:
  - [ ] 학습 시간, 목표 달성률, 출석률 등 데이터 시각화
  - [ ] 그룹 내 개인 랭킹, 전체 사용자 대비 백분위 등 비교 데이터 제공
- [ ] **게이미피케이션**:
  - [ ] 활동에 따른 포인트, 배지, 레벨 시스템
  - [ ] 스터디 목표 달성, 연속 출석 등 챌린지 퀘스트

### 2.4. 플랫폼 확장

- [ ] **멘토링 시스템**: 특정 분야의 전문가(멘토)와 학습자(멘티)를 연결하는 기능
- [ ] **B2B 모델**: 기업 연계 채용 연계형 스터디나 과제 챌린지 진행
- [ ] **모바일 앱**: 모바일 환경에 최적화된 네이티브 앱(iOS, Android) 개발
