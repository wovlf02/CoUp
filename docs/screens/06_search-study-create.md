## 🎨 섹션별 상세 설계

### 1. 페이지 헤더
```
← 뒤로가기

새 스터디 만들기
함께 성장할 멤버를 모집하세요
```

**뒤로가기**:
- 아이콘 버튼 (text-sm, gray-600)
- 클릭 → 이전 페이지 또는 `/studies`

**제목**:
- "새 스터디 만들기" (text-2xl, Bold, gray-900)
- "함께 성장할 멤버를 모집하세요" (text-sm, gray-600)

### 2. 폼 섹션 1: 기본 정보
```
┌───────────────────────────────────────────────────────────┐
│ 1. 기본 정보                                              │
│                                                           │
│ 스터디 이름 *                                             │
│ ┌───────────────────────────────────────────────────────┐│
│ │ 코딩테스트 마스터 스터디                               ││
│ └───────────────────────────────────────────────────────┘│
│ 2~50자 (현재: 13자)                                       │
│                                                           │
│ 카테고리 *                                                │
│ ┌──────────────────┐  ┌──────────────────┐              │
│ │ 프로그래밍  ▼     │  │ 알고리즘/코테 ▼  │              │
│ └──────────────────┘  └──────────────────┘              │
│                                                           │
│ 스터디 소개 *                                             │
│ ┌───────────────────────────────────────────────────────┐│
│ │ 매일 아침 알고리즘 문제를 풀고                         ││
│ │ 서로의 풀이를 공유합니다.                              ││
│ │ 함께 실력을 키워나가요!                                ││
│ │                                                        ││
│ └───────────────────────────────────────────────────────┘│
│ 10~500자 (현재: 45자)                                     │
│                                                           │
│ 태그 (선택)                                               │
│ ┌───────────────────────────────────────────────────────┐│
│ │ #알고리즘 × #코딩테스트 × #매일                        ││
│ │ + 태그 추가                                            ││
│ └───────────────────────────────────────────────────────┘│
│ 최대 5개                                                  │
└───────────────────────────────────────────────────────────┘
```

**필드 설명**:

1. **스터디 이름** (필수)
   - Input: text, maxLength=50
   - 실시간 글자 수 표시
   - 검증: 2~50자
   - 에러: "2자 이상 입력해주세요"

2. **카테고리** (필수)
   - 메인 카테고리: Dropdown
     - 프로그래밍, 취업준비, 자격증, 어학, 운동, 독서, 기타
   - 서브 카테고리: Dropdown (동적)
     - 프로그래밍 → 웹 개발, 앱 개발, 알고리즘/코테, AI/ML 등

3. **스터디 소개** (필수)
   - Textarea: rows=6, maxLength=500
   - 실시간 글자 수 표시
   - 검증: 10~500자

4. **태그** (선택)
   - Input + 엔터로 추가
   - 최대 5개
   - X 버튼으로 삭제
   - 스타일: Primary-100 배경, Primary-700 텍스트

### 3. 폼 섹션 2: 모집 설정
```
┌───────────────────────────────────────────────────────────┐
│ 2. 모집 설정                                              │
│                                                           │
│ 모집 인원 *                                               │
│ ┌───┐  ┌────┐  ┌───┐                                     │
│ │ - │  │ 10 │  │ + │  명                                 │
│ └───┘  └────┘  └───┘                                     │
│ 최소 2명, 최대 50명                                        │
│                                                           │
│ 공개 설정 *                                               │
│ (•) 전체 공개                                             │
│     누구나 검색하고 가입할 수 있습니다                      │
│                                                           │
│ ( ) 비공개                                                │
│     초대받은 사람만 가입할 수 있습니다                      │
│                                                           │
│ 자동 승인                                                 │
│ [✓] 가입 신청 시 자동으로 승인                            │
│ (체크 해제 시 그룹장이 수동 승인)                          │
└───────────────────────────────────────────────────────────┘
```

**필드 설명**:

1. **모집 인원** (필수)
   - Number Input + 증가/감소 버튼
   - 범위: 2~50명
   - 기본값: 10명

2. **공개 설정** (필수)
   - Radio Button
   - 전체 공개 (PUBLIC) - 기본값
   - 비공개 (PRIVATE)

3. **자동 승인** (선택)
   - Checkbox
   - 기본값: true (자동 승인)

### 4. 하단 액션 버튼
```
                    [취소]  [스터디 만들기]
```

**취소 버튼** (Secondary):
- 클릭 → 확인 다이얼로그
- "작성 중인 내용이 사라집니다. 계속하시겠습니까?"
- 확인 → `/studies`

**스터디 만들기** (Primary):
- 필수 항목 미입력 시 비활성화
- 클릭 → 로딩 스피너
- 성공 → Toast "스터디가 생성되었습니다!"
- 스터디 상세 페이지로 이동

## 🎬 인터랙션

### 실시간 검증
```javascript
// 스터디 이름 검증
if (name.length < 2) {
  error = "2자 이상 입력해주세요"
} else if (name.length > 50) {
  error = "50자를 초과할 수 없습니다"
}

// 소개 검증
if (description.length < 10) {
  error = "10자 이상 입력해주세요"
}

// 태그 검증
if (tags.length > 5) {
  error = "태그는 최대 5개까지 추가할 수 있습니다"
}
```

### 제출 흐름
```
1. 버튼 클릭
2. 클라이언트 검증
   - 필수 항목 확인
   - 글자 수 제한 확인
3. 로딩 상태 (버튼 disabled + 스피너)
4. API 요청 (POST /api/v1/studies)
5. 성공
   - Toast 알림: "스터디가 생성되었습니다!"
   - 리다이렉트 → /studies/[새로운ID]
6. 실패
   - 에러 메시지 표시 (필드 아래 또는 Toast)
   - 예: "이미 같은 이름의 스터디가 존재합니다"
```
