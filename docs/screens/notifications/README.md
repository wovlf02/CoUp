# 알림(Notifications) 기능 설계

> **네비게이션 탭**: 🔔 알림  
> **기본 라우트**: `/notifications`  
> **목적**: 스터디 활동 알림을 실시간으로 수신 및 관리

---

## 📌 주요 기능

### 1. 실시간 알림 수신
- WebSocket을 통한 실시간 푸시
- 브라우저 Toast 알림
- 헤더 배지 카운트 업데이트

### 2. 알림 유형
- **가입승인**: 스터디 가입 승인/거절
- **공지**: 새 공지사항 등록
- **파일**: 새 파일 업로드
- **일정**: 새 일정 또는 임박한 일정 알림
- **할일**: 할 일 추가/완료
- **멤버**: 새 멤버 가입/퇴장
- **강퇴**: 스터디에서 강퇴됨
- **댓글**: 내 게시물에 댓글
- **멘션**: 채팅/댓글에서 나를 멘션

### 3. 필터링
- **전체**: 모든 알림
- **읽지않음**: 읽지 않은 알림만 (기본)

### 4. 읽음 처리
- 알림 클릭 시 자동 읽음 처리
- 해당 페이지로 자동 이동
- [모두 읽음] 버튼으로 일괄 처리

### 5. 알림 목록
- 최신순 정렬
- 읽지 않은 알림: 파란 점 표시 + 강조 배경
- 읽은 알림: 일반 배경
- 페이지네이션 (20개씩)

## 🎨 레이아웃 특징

### 2컬럼 구조
- 좌측 네비게이션: 12% (240px)
- 메인 콘텐츠: 88%

### 알림 카드
- 읽지 않음: 파란색 배경 + 파란 점
- 읽음: 흰색 배경 + 흰 점
- 유형별 배지 색상

---

## 🎯 클릭 동작

| 알림 유형 | 이동 경로 |
|----------|----------|
| 가입승인 | `/my-studies/[studyId]` |
| 공지 | `/my-studies/[studyId]/notices/[noticeId]` |
| 파일 | `/my-studies/[studyId]/files` |
| 일정 | `/my-studies/[studyId]/calendar` |
| 할일 | `/tasks` |
| 멤버 | `/my-studies/[studyId]/members` |
| 댓글 | 원본 게시물 |
| 멘션 | 해당 채팅/댓글 |
