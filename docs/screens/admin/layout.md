# 화면 설계 - 레이아웃 및 네비게이션

> **파일**: layout.md
> **목적**: 모든 관리자 페이지에 일관되게 적용되는 기본 레이아웃과 네비게이션 구조를 정의합니다.

---

## 1. 기본 레이아웃 (Shell)

관리자 시스템의 모든 페이지는 '앱 셸(App Shell)'이라고 불리는 공통적인 레이아웃 구조를 공유합니다. 이는 사용자가 페이지를 이동하더라도 일관된 경험을 제공하며, 핵심 네비게이션에 항상 접근할 수 있도록 보장합니다.

### 1.1 구조 다이어그램

```
+----------------------------------------------------------------------+
| AdminHeader (높이: 64px)                                             |
| (페이지 제목, 검색창, 알림 아이콘, 사용자 프로필)                    |
+--------------------------+-------------------------------------------+
|                          |                                           |
| AdminSidebar (너비: 240px)|                                           |
| (메인 네비게이션 메뉴)   |   Main Content Area (나머지 공간)         |
|                          |   (각 페이지의 실제 내용)                 |
|                          |                                           |
|                          |                                           |
|                          |                                           |
|                          |                                           |
|                          |                                           |
+--------------------------+-------------------------------------------+
```

-   **AdminHeader**: 페이지 상단에 고정되어 현재 페이지의 제목을 표시하고, 전역 검색, 알림, 프로필 관리 등 빠른 액션을 제공합니다.
-   **AdminSidebar**: 좌측에 고정되어 시스템의 주요 기능 영역으로 이동할 수 있는 메인 네비게이션 메뉴를 제공합니다.
-   **Main Content Area**: 실제 페이지의 콘텐츠가 렌더링되는 영역입니다. 이 영역은 필요에 따라 스크롤됩니다.

### 1.2 UX 원칙
- **일관성**: 어느 페이지에 있든 헤더와 사이드바의 위치와 기능은 동일하게 유지됩니다.
- **예측 가능성**: 사용자는 메뉴 항목을 보고 어떤 내용이 나올지 쉽게 예측할 수 있습니다.
- **효율성**: 자주 사용하는 기능(검색, 프로필)은 헤더에, 주요 기능 영역은 사이드바에 배치하여 최소한의 클릭으로 접근할 수 있도록 합니다.

---

## 2. AdminHeader 상세 설계

- **구성 요소**:
    - **페이지 제목 (`h1`)**: 현재 페이지의 이름을 명확하게 표시합니다. (예: '대시보드', '사용자 관리')
    - **전역 검색창 (선택 사항)**: 사용자, 스터디 등 시스템 전반의 데이터를 빠르게 검색할 수 있는 입력창입니다.
    - **알림 아이콘 (종 모양)**: 읽지 않은 중요 알림이 있을 경우 뱃지(badge)를 표시합니다. 클릭 시 알림 목록 드롭다운이 나타납니다.
    - **사용자 프로필**: 현재 로그인한 관리자의 프로필 이미지와 이름을 표시합니다. 클릭 시 '내 정보', '로그아웃' 등의 메뉴가 포함된 드롭다운이 나타납니다.

---

## 3. AdminSidebar 상세 설계

- **구성 요소**:
    - **로고**: 최상단에 CoUp 로고를 표시하며, 클릭 시 메인 대시보드로 이동합니다.
    - **네비게이션 그룹**: 관련 있는 메뉴 항목들을 그룹으로 묶어 시각적으로 구분합니다. (예: '주요 기능', '분석', '설정')
    - **메뉴 항목**: 아이콘과 텍스트로 구성됩니다. 현재 활성화된 페이지의 메뉴 항목은 배경색이나 텍스트 색상으로 강조 표시하여 사용자가 현재 위치를 명확히 알 수 있도록 합니다.
- **네비게이션 메뉴 목록**:
    - 📊 **대시보드** (`/admin/dashboard`)
    - 👥 **사용자 관리** (`/admin/users`)
    - 📚 **스터디 관리** (`/admin/studies`)
    - 🚨 **신고 관리** (`/admin/reports`)
    - 🛡️ **콘텐츠 모더레이션** (`/admin/moderation`)
    - 📈 **분석** (`/admin/analytics`)
    - ⚙️ **시스템 설정** (`/admin/settings`) - `SYSTEM_ADMIN`에게만 노출

### 3.1 인터랙션 (UX)
- **호버**: 메뉴 항목에 마우스를 올리면 배경색이 부드럽게 변경되어 클릭 가능한 요소임을 시각적으로 암시합니다.
- **활성화 상태**: 현재 페이지와 일치하는 메뉴 항목은 명확하게 강조 표시되어야 합니다.
- **접기/펼치기 (선택 사항)**: 화면 공간을 더 확보하고 싶을 때를 대비해, 사이드바를 아이콘만 보이는 좁은 형태로 접을 수 있는 기능을 제공할 수 있습니다.

---

## 4. 반응형 설계

- **데스크톱 (1280px 이상)**: 헤더와 사이드바가 모두 고정된 상태로 표시됩니다.
- **태블릿 (768px ~ 1279px)**: 사이드바가 기본적으로 접혀있고, 헤더의 햄버거 메뉴 아이콘을 클릭하여 펼칠 수 있습니다. (오버레이 방식 또는 콘텐츠 영역을 밀어내는 방식)
- **모바일 (767px 이하)**: 사이드바는 항상 숨겨져 있으며, 햄버거 메뉴를 통해서만 접근할 수 있습니다.
