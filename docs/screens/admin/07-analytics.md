# 화면 설계 - 07: 분석

> **파일**: 07-analytics.md
> **경로**: `/admin/analytics`
> **목적**: 관리자와 운영자가 플랫폼의 성장 추이와 사용자 행동 패턴을 데이터 기반으로 파악하고, 비즈니스 전략 수립에 필요한 인사이트를 얻을 수 있도록 합니다.

---

## 1. 분석 대시보드 페이지 (`/admin/analytics`)

### 1.1 레이아웃 및 구성 요소

분석 대시보드는 여러 하위 분석 페이지로 연결되는 허브 역할을 하며, 가장 중요한 KPI들을 요약하여 보여줍니다.

```
+----------------------------------------------------------------------+
| AdminHeader (페이지 제목: "분석 대시보드")                           |
+----------------------------------------------------------------------+
| TabNavigation (사용자 분석, 스터디 분석, 신고 분석 등)               |
+----------------------------------------------------------------------+
| 전역 필터: [기간 선택: 지난 30일 ▼]                                  |
+----------------------------------------------------------------------+
|                                                                      |
|  (선택된 탭에 해당하는 상세 분석 내용)                               |
|                                                                      |
+----------------------------------------------------------------------+
```

-   **TabNavigation**: 분석의 대주제(사용자, 스터디 등)를 선택하는 탭. 각 탭은 하위 경로(예: `/admin/analytics/users`)에 해당합니다.
-   **전역 필터**: 페이지 전체에 적용되는 기간 필터. 여기서 기간을 '지난 90일'로 변경하면 페이지 내 모든 차트와 데이터가 해당 기간에 맞게 업데이트됩니다.

## 2. 사용자 분석 탭 (`/admin/analytics/users`)

### 2.1 구성 요소
1.  **핵심 사용자 지표 카드**: 선택된 기간의 MAU(월간 활성 사용자), 신규 가입자 수, 재방문율 등을 표시.
2.  **사용자 성장 그래프**: 시간에 따른 누적 가입자 수와 신규 가입자 수 추이를 보여주는 라인/바 차트.
3.  **코호트 분석 (재방문율) 테이블**:
    -   **행**: 가입일 기준 사용자 그룹 (예: '11월 1주차 가입자')
    -   **열**: 가입 후 경과 시간 (1주차, 2주차, ... , 8주차)
    -   **값**: 각 주차에 재방문한 사용자의 비율(%).
    -   **UX**: 각 셀의 비율이 높을수록 진한 색상으로 표시(Heatmap)하여, 어떤 코호트의 만족도가 높은지 직관적으로 파악할 수 있게 합니다.

---

## 3. 스터디 분석 탭 (`/admin/analytics/studies`)

### 3.1 구성 요소
1.  **카테고리별 분포 차트**: 어떤 분야의 스터디가 가장 많이 생성되고 활동하는지 보여주는 도넛 차트 또는 파이 차트.
    -   **UX**: 차트의 특정 카테고리 조각을 클릭하면, 아래의 스터디 목록이 해당 카테고리로 필터링됩니다.
2.  **품질 분포 차트**: 전체 스터디를 '우수(Excellent)', '보통(Good)', '저품질(Poor)' 그룹으로 나누어 그 비중을 보여주는 바 차트.
    -   **UX**: 운영 목표(예: 저품질 스터디 10% 미만 유지) 대비 현재 상태를 함께 보여주어, 운영 상태가 양호한지 직관적으로 알 수 있게 합니다.
3.  **신규 스터디 생성 추이 그래프**: 시간에 따른 신규 스터디 개설 수를 보여주는 라인 차트.

---

## 4. 신고 분석 탭 (`/admin/analytics/reports`)

### 4.1 구성 요소
1.  **처리 현황 통계 카드**: 선택된 기간의 총 접수 건수, 처리 완료 건수, 평균 처리 시간(Time to Resolution)을 표시.
2.  **신고 유형별 분포 차트**: '스팸', '괴롭힘', '부적절 콘텐츠' 등 어떤 유형의 신고가 가장 많은지 보여주는 파이 차트.
    -   **UX**: 특정 유형을 클릭하면, 해당 유형의 신고가 어떤 카테고리(사용자, 스터디, 메시지)를 대상으로 하는지 세부 내역을 보여주는 드릴다운(Drill-down) 기능을 제공합니다.
3.  **담당자별 처리 현황 테이블**: 각 관리자가 몇 건의 신고를 처리했는지, 평균 처리 시간은 얼마인지 등을 비교하여 보여주는 테이블. 이는 팀의 성과를 측정하고 업무량을 조절하는 데 사용됩니다.

---

## 5. 리포트 생성 및 내보내기

-   **기능**: 각 분석 페이지 우측 상단에 '[리포트 생성]' 또는 '[CSV 내보내기]' 버튼을配置합니다.
-   **UX**:
    -   '[CSV 내보내기]'를 클릭하면 현재 화면에 보이는 필터링된 데이터를 기반으로 CSV 파일을 즉시 다운로드합니다.
    -   '[리포트 생성]'을 클릭하면, 현재 페이지의 차트와 데이터를 포함한 PDF 형식의 보고서를 생성하여 다운로드하거나 다른 관리자에게 이메일로 전송할 수 있는 옵션을 제공합니다.
