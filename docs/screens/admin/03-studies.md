# 화면 설계 - 03: 스터디 관리

> **파일**: 03-studies.md
> **경로**: `/admin/studies`, `/admin/studies/[studyId]`
> **목적**: 관리자가 플랫폼의 모든 스터디를 모니터링하고, 품질을 관리하며, 문제가 있는 스터디에 대해 적절한 조치를 취할 수 있도록 합니다.

---

## 1. 스터디 목록 페이지 (`/admin/studies`)

### 1.1 레이아웃 및 구성 요소

```
+----------------------------------------------------------------------+
| AdminHeader (페이지 제목: "스터디 관리")                             |
+----------------------------------------------------------------------+
| TabNavigation (전체, 활성, 저품질, 추천, 신고됨)                     |
+----------------------------------------------------------------------+
| FilterPanel (카테고리, 멤버 수 범위 등) | SearchBar (스터디 이름 검색) |
+----------------------------------------------------------------------+
|                                                                      |
|  StudiesTable (스터디 목록 데이터 테이블)                            |
|                                                                      |
+----------------------------------------------------------------------+
| Pagination (페이지 이동)                                             |
+----------------------------------------------------------------------+
```

-   **TabNavigation**: 미리 정의된 필터(활성, 저품질 등)로 목록을 빠르게 전환하는 탭 UI.
-   **FilterPanel & SearchBar**: 세부 조건으로 스터디를 필터링하고 검색하는 UI.
-   **StudiesTable**: 스터디 목록을 표시하는 테이블.

### 1.2 UX (사용자 흐름 및 인터랙션)

1.  **탭 기반 필터링**:
    -   페이지에 처음 진입 시 '전체' 탭이 기본으로 활성화됩니다.
    -   관리자가 '저품질' 탭을 클릭하면, URL이 `?tab=low-quality`로 변경되고, API를 재호출하여 품질 점수가 40점 미만인 스터디 목록만 테이블에 표시합니다. 이를 통해 관리자는 조치가 필요한 스터디 그룹에 빠르게 집중할 수 있습니다.
2.  **상세 정보 확인**:
    -   `StudiesTable`의 특정 스터디 행을 클릭하면, 상세 페이지(`/admin/studies/[studyId]`)로 이동합니다.
3.  **빠른 액션**:
    -   테이블의 각 행 끝에는 '추천 설정', '공개/비공개 전환' 등 빠른 조치를 위한 버튼이 있습니다.
    -   '추천 설정'이 불가능한 스터디(자격 미달)의 경우, 버튼을 비활성화하고 마우스를 올리면 "추천 자격 미달: 품질 점수 부족"과 같은 사유를 툴팁으로 보여주어 직관적인 피드백을 제공합니다.

## 2. 스터디 상세 페이지 (`/admin/studies/[studyId]`)

### 2.1 레이아웃 및 구성 요소

```
+-------------------------------------------------------------------------+
| AdminHeader (페이지 제목: "{스터디 이름}")                              |
+--------------------------------+----------------------------------------+
|                                |                                        |
| StudyProfileCard (좌측, 30%)   |  Tab-based Info (우측, 70%)            |
| (스터디 기본 정보, 조치 버튼)  |  (기본 정보, 멤버, 활동 로그 등)       |
|                                |                                        |
|                                |                                        |
| QualityReportCard              |                                        |
| (품질 점수 상세 내역)          |                                        |
|                                |                                        |
+--------------------------------+----------------------------------------+
```

-   **StudyProfileCard**: 스터디의 대표 이미지, 이름, 설명 등 기본 정보와 '추천 설정', '삭제' 등 주요 액션 버튼을 포함합니다.
-   **QualityReportCard**: 현재 품질 점수와 각 항목별(활동성, 충원율 등) 세부 점수를 보여주어, 관리자가 왜 이 점수가 나왔는지 명확히 이해하도록 돕습니다.
-   **Tab-based Info**: 상세 정보를 여러 탭으로 나누어 제공합니다.

### 2.2 UX (사용자 흐름 및 인터랙션)

1.  **품질 점수 확인**:
    -   관리자는 `QualityReportCard`를 보고 "최근 활동이 저조하여 활동성 점수가 낮음"과 같은 문제의 원인을 즉시 파악할 수 있습니다.
2.  **멤버 관리**:
    -   '멤버' 탭에서 스터디에 참여 중인 멤버 목록을 확인합니다.
    -   소유자(LEADER)가 장기 비활동 상태일 경우, `SYSTEM_ADMIN`은 다른 멤버에게 소유자 권한을 위임할 수 있는 버튼을 볼 수 있습니다. 이 버튼을 클릭하면 확인 모달이 나타납니다.
3.  **스터디 모더레이션**:
    -   `StudyProfileCard`의 '삭제' 버튼(SYSTEM_ADMIN 전용)을 클릭하면, "이 스터디와 관련된 모든 데이터가 삭제됩니다. 정말 삭제하시겠습니까?"와 같이 위험성을 경고하는 확인 모달이 나타납니다.
    -   '비공개 전환' 버튼을 클릭하면, 즉시 API가 호출되고 버튼은 '공개로 전환'으로 바뀝니다(Optimistic UI). 동시에 화면 우측 상단에 "스터디가 비공개 처리되었습니다."라는 토스트 알림이 나타나 작업이 완료되었음을 명확히 알려줍니다.
