# 화면 설계 - 01: 대시보드

> **파일**: 01-dashboard.md
> **경로**: `/admin/dashboard`
> **목적**: 관리자가 시스템의 전반적인 현황을 한눈에 파악하고, 시급한 문제에 빠르게 대응할 수 있도록 핵심 지표와 정보를 시각적으로 제공합니다.

---

## 1. 레이아웃 및 구성 요소

대시보드는 관리자가 가장 먼저 마주하는 페이지로서, 정보의 중요도와 시급성에 따라 그리드(Grid) 시스템 기반으로 배치됩니다.

### 1.1 구조 다이어그램

```
+----------------------------------------------------------------------+
| AdminHeader (페이지 제목: "대시보드")                                |
+--------------------------+----------------------+--------------------+
| StatCard: 총 사용자      | StatCard: 활성 스터디| StatCard: 처리 대기 신고 |
+--------------------------+----------------------+--------------------+
| StatCard: 오늘 DAU       | 기간 선택: [지난 7일 ▼] |                      |
+--------------------------+-------------------------------------------+
|                                                                      |
|  ActivityChart: 주간 사용자 활동/신규 가입자 추이 (75%)              |
|                                                                      |
|                                                                      |
+----------------------------------------------------------------------+
|                                                                      |
|  RecentActivityFeed: 최근 활동 (50%) | AlertsList: 긴급 알림 (50%)   |
|                                     |                              |
|                                     |                              |
+----------------------------------------------------------------------+
```

### 1.2 핵심 컴포넌트
1.  **StatCard (통계 카드)**: 4개의 주요 KPI(총 사용자, 활성 스터디, 처리 대기 신고, 일일 활성 사용자)를 표시하는 카드.
2.  **ActivityChart (활동 차트)**: 특정 기간(기본 7일) 동안의 사용자 활동 추이를 보여주는 라인 차트.
3.  **RecentActivityFeed (최근 활동 피드)**: 신규 스터디 생성, 관리자 임명 등 시스템의 주요 이벤트가 시간순으로 표시되는 목록.
4.  **AlertsList (긴급 알림 목록)**: 우선순위가 'URGENT' 또는 'HIGH'인 미처리 신고 목록을 표시하여 빠른 대응을 유도.

## 2. 컴포넌트 상세 설계 및 UX

### 2.1 StatCard (통계 카드)

-   **표시 정보**:
    -   **제목**: 지표의 이름 (예: "총 사용자")
    -   **수치**: 현재 값 (예: "15,230명")
    -   **아이콘**: 지표를 상징하는 아이콘 (예: 사용자 아이콘)
    -   **변동률 (선택 사항)**: 전일/전주 대비 증감률을 화살표(▲, ▼)와 함께 표시하여 변화 추이를 직관적으로 인지시킵니다. (예: `+1.5%`)
-   **UX (인터랙션)**:
    -   카드에 마우스를 올리면(hover) 그림자 효과가 살짝 강조되어 클릭 가능한 요소임을 암시합니다.
    -   카드를 클릭하면 해당 지표의 상세 분석 페이지로 이동합니다. (예: '총 사용자' 카드 클릭 시 `/admin/analytics/users`로 이동)

### 2.2 ActivityChart (활동 차트)

-   **기본 뷰**: 최근 7일간의 일일 활성 사용자(DAU)와 신규 가입자 수를 두 개의 라인으로 표시합니다.
-   **UX (인터랙션)**:
    -   차트 상단의 '기간 선택' 드롭다운(지난 7일, 지난 30일, 지난 3개월)을 통해 조회 기간을 변경할 수 있습니다.
    -   차트의 특정 데이터 포인트에 마우스를 올리면 툴팁(Tooltip)이 나타나 해당 날짜의 정확한 수치를 보여줍니다.
    -   차트 하단의 범례(Legend)를 클릭하여 특정 데이터(예: '신규 가입자') 라인을 숨기거나 다시 표시할 수 있습니다.

### 2.3 RecentActivityFeed (최근 활동 피드)

-   **표시 정보**:
    -   각 항목은 아이콘, 활동 내용, 발생 시간으로 구성됩니다.
    -   예: `[스터디 생성 아이콘] "React 초급 스터디"가 생성되었습니다. (2시간 전)`
    -   예: `[관리자 아이콘] "김관리"님이 "이사용"님을 정지시켰습니다. (3시간 전)`
-   **UX (인터랙션)**:
    -   피드의 각 항목은 관련된 상세 페이지로 이동하는 링크를 가집니다. (예: 스터디 이름을 클릭하면 해당 스터디 상세 페이지로 이동)
    -   피드 하단에 '더 보기' 버튼을 두어 전체 활동 로그 페이지로 이동시킵니다.

### 2.4 AlertsList (긴급 알림 목록)

-   **표시 정보**:
    -   처리 대기 중인 신고 중 우선순위가 'URGENT' 또는 'HIGH'인 항목만 최대 5개까지 표시합니다.
    -   각 항목은 우선순위(색상으로 구분), 신고 유형, 대상, 신고 시간으로 구성됩니다.
    -   예: `[URGENT] 괴롭힘 신고 | 대상: 박악플러 (30분 전)`
-   **UX (인터랙션)**:
    -   각 항목을 클릭하면 해당 신고의 상세 처리 페이지(`/admin/reports/:id`)로 즉시 이동하여 문제를 빠르게 처리할 수 있도록 동선을 최소화합니다.
    -   목록 하단에 '모든 신고 보기' 링크를 제공하여 전체 신고 목록 페이지로 이동시킵니다.

## 3. 데이터 페칭 전략

-   대시보드는 여러 종류의 데이터를 한 번에 보여주므로, 데이터 로딩으로 인한 성능 저하를 최소화해야 합니다.
-   **초기 렌더링**: `StatCard`와 같이 필수적인 핵심 지표는 페이지와 함께 서버에서 렌더링합니다.
-   **스트리밍 (Streaming)**: `ActivityChart`, `RecentActivityFeed`와 같이 상대적으로 무겁거나 덜 중요한 컴포넌트는 React `Suspense`를 사용하여 스트리밍합니다. 이를 통해 사용자는 핵심 콘텐츠를 먼저 보고, 나머지 데이터가 로드되는 동안 로딩 스켈레톤 UI를 보게 되어 체감 로딩 속도가 향상됩니다.
-   **캐싱**: 대시보드 데이터는 실시간일 필요는 없으므로, 각 데이터 소스 API에 1분~5분 정도의 짧은 캐시(Redis 또는 Next.js fetch cache)를 적용하여 DB 부하를 줄입니다.
