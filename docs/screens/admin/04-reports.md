# 화면 설계 - 04: 신고 관리

> **파일**: 04-reports.md
> **경로**: `/admin/reports`, `/admin/reports/[reportId]`
> **목적**: 사용자가 제출한 신고를 관리자가 체계적으로 검토하고, 신속하고 정확하게 처리할 수 있는 워크플로우를 제공합니다.

---

## 1. 신고 목록 페이지 (`/admin/reports`)

### 1.1 레이아웃 및 구성 요소

```
+----------------------------------------------------------------------+
| AdminHeader (페이지 제목: "신고 관리")                               |
+----------------------------------------------------------------------+
| FilterPanel (상태, 우선순위, 유형, 담당자 등)                        |
+----------------------------------------------------------------------+
|                                                                      |
|  ReportsList (카드 또는 테이블 형태의 신고 목록)                     |
|                                                                      |
|                                                                      |
+----------------------------------------------------------------------+
| Pagination (페이지 이동)                                             |
+----------------------------------------------------------------------+
```

-   **FilterPanel**: 처리 상태(대기, 진행중, 완료), 우선순위(긴급, 높음 등), 신고 유형 등 다양한 조건으로 신고 목록을 필터링합니다. '내게 할당된 신고' 필터는 관리자가 자신의 업무에 집중하도록 돕습니다.
-   **ReportsList**: 각 신고를 요약 정보(우선순위, 유형, 대상, 시간)가 담긴 '카드' 형태로 표시합니다. 카드는 우선순위에 따라 배경색이나 좌측 테두리 색상(긴급: 빨강, 높음: 주황 등)이 달라져 시각적 구분이 용이합니다.

### 1.2 UX (사용자 흐름 및 인터랙션)

1.  **우선순위 인지**: 관리자는 페이지에 진입하자마자 색상으로 구분된 '긴급(URGENT)' 신고 카드를 즉시 인지하고 먼저 처리할 수 있습니다.
2.  **신속한 처리**: 각 카드에는 '상세 보기' 버튼 외에, '승인(문제 있음)', '거절(문제 없음)'과 같은 빠른 처리 버튼을 제공할 수 있습니다. 관리자가 신고 내용을 요약 정보만으로 판단할 수 있을 때, 이 버튼을 클릭하면 상세 페이지에 들어가지 않고도 즉시 신고를 처리할 수 있습니다.
3.  **상세 검토**: '상세 보기' 버튼이나 카드 자체를 클릭하면, 화면 우측에 해당 신고의 상세 정보를 담은 사이드 패널이 나타납니다. 페이지를 벗어나지 않고 목록과 상세 내용을 동시에 볼 수 있어 컨텍스트 전환 비용이 줄어듭니다.

---

## 2. 신고 상세 (사이드 패널)

사이드 패널은 3단 구조로 구성되어, 관리자가 신고를 처리하는 데 필요한 모든 정보를 한 화면에서 확인할 수 있도록 UX를 극대화합니다.

### 2.1 레이아웃 및 구성 요소

```
+------------------------------------------------+
| PanelHeader                                    |
| (신고 ID, 우선순위, 상태, 담당자)              |
+------------------------------------------------+
|                                                |
| 1. 신고된 콘텐츠 (상단)                        |
|    (채팅 메시지, 게시글 등 원본 콘텐츠)        |
|                                                |
+------------------------------------------------+
|                                                |
| 2. 증거 및 이력 (중단)                         |
|    (신고자 코멘트, 스크린샷, 피신고자 제재 이력)|
|                                                |
+------------------------------------------------+
|                                                |
| 3. 처리 액션 (하단)                            |
|    (승인/거절/보류, 제재 선택, 사유 입력)      |
|                                                |
+------------------------------------------------+
```

1.  **신고된 콘텐츠**: 문제의 소지가 있는 콘텐츠 원본을 직접 보여주어 관리자가 컨텍스트를 정확히 파악하게 합니다.
2.  **증거 및 이력**: 신고자가 제출한 증거(설명, 이미지)와, 시스템이 제공하는 피신고자의 과거 제재 이력을 나란히 보여주어 관리자가 종합적인 판단을 내리도록 돕습니다.
3.  **처리 액션**: 관리자가 판단을 내린 후, 실제 조치를 취하는 영역입니다.

### 2.2 UX (사용자 흐름 및 인터랙션)

1.  **정보 확인**: 관리자는 스크롤을 통해 신고 원본, 증거, 피신고자 이력을 순차적으로 확인합니다.
2.  **AI 추천 조치**: 피신고자의 과거 이력과 신고 유형을 바탕으로, 시스템이 "💡 추천 조치: 7일 정지 (유사 위반 2회)" 와 같은 제재를 추천해줍니다. 이는 관리자의 판단을 돕고 처리 기준의 일관성을 높입니다.
3.  **처리 결정**:
    -   '승인' 라디오 버튼을 선택하면, 아래에 '제재 조치 선택' 드롭다운이 활성화됩니다.
    -   '거절'을 선택하면, '제재 조치' 드롭다운은 비활성화됩니다.
4.  **사유 입력 및 완료**: 관리자가 처리 사유를 입력하고 '처리 완료' 버튼을 누릅니다.
5.  **자동 업데이트**: 처리가 완료되면 사이드 패널은 자동으로 닫히고, 좌측의 신고 목록에서 해당 신고 카드의 상태가 '처리 완료'로 변경되며 흐릿하게 처리됩니다(Optimistic UI). 동시에 "신고 #12345 처리가 완료되었습니다." 라는 토스트 알림이 표시됩니다.
