# MVP 기능 명세: 02. 스터디 그룹 관리

## 1. 개요

- 사용자는 스터디 그룹을 생성하고, 다른 사용자가 만든 스터디 그룹을 탐색하고 가입할 수 있다.
- 그룹장은 그룹 정보와 멤버를 관리할 수 있다.

## 2. 사용자 스토리 (User Stories)

### 2.1. 그룹 생성 (CRUD - Create)

- **US-1.1**: 로그인한 사용자는 '스터디 생성' 페이지로 이동할 수 있다.
- **US-1.2**: 사용자는 스터디 그룹 생성을 위해 다음 정보를 입력할 수 있다.
  - 스터디 이름 (필수, 중복 가능)
  - 스터디 소개 (필수, Markdown 지원)
  - 스터디 목표 (선택, 텍스트)
  - 스터디 카테고리 (필수, 예: `프로그래밍`, `수능`, `어학`)
  - 스터디 규칙 (선택, 텍스트 에어리어)
  - 공개 여부 (필수, `공개` 또는 `비공개`)
  - 최대 정원 (선택, 기본값 10명)
- **US-1.3**: 그룹을 생성한 사용자는 자동으로 해당 그룹의 `OWNER`(그룹장)가 된다.

### 2.2. 그룹 탐색 및 조회 (CRUD - Read)

- **US-2.1**: 모든 사용자는 '스터디 탐색' 페이지에서 현재 모집 중인 `공개` 스터디 그룹 목록을 볼 수 있다.
- **US-2.2**: 스터디 그룹 목록은 무한 스크롤 또는 페이지네이션으로 제공된다.
- **US-2.3**: 사용자는 키워드를 통해 스터디 이름 또는 소개 내용으로 그룹을 검색할 수 있다.
- **US-2.4**: 사용자는 카테고리별로 스터디 그룹을 필터링하여 볼 수 있다.
- **US-2.5**: 사용자는 목록의 특정 스터디 그룹을 클릭하여 상세 페이지로 이동할 수 있다.
- **US-2.6**: 그룹 상세 페이지에서는 그룹 소개, 스터디 목표, 현재 인원/최대 정원, 멤버 목록, 그룹 규칙 등을 확인할 수 있다.

### 2.3. 그룹 정보 수정 및 삭제 (CRUD - Update, Delete)

- **US-3.1**: 그룹장(`OWNER`)은 자신이 생성한 스터디 그룹의 정보를 수정할 수 있다. (이름, 소개, 카테고리, 스터디 규칙 등)
- **US-3.2**: 그룹장(`OWNER`)은 스터디 그룹을 삭제할 수 있다. 그룹 삭제 시, 모든 멤버와 활동 내역에 대한 처리 방침을 명확히 해야 한다. (예: 논리적 삭제)

### 2.4. 그룹 참여 및 멤버 관리

- **US-4.1**: 로그인한 사용자는 `공개` 스터디 그룹의 상세 페이지에서 '가입 신청' 버튼을 누를 수 있다. 이때, 그룹장에게 보낼 간단한 자기소개 또는 가입 동기 메시지를 작성할 수 있다.
- **US-4.2**: `비공개` 스터디 그룹은 초대를 통해서만 가입할 수 있다. (초대 기능은 Post-MVP)
- **US-4.3**: 그룹장은 '그룹 관리' 페이지에서 가입 신청 목록을 확인하고, 신청자의 프로필 정보와 가입 메시지를 검토한 후 신청을 `승인`하거나 `거절`할 수 있다.
- **US-4.4**: 가입이 승인된 사용자는 그룹의 `MEMBER`가 된다.
- **US-4.5**: 그룹 멤버는 스스로 그룹을 `탈퇴`할 수 있다.
- **US-4.6**: 그룹장은 그룹 멤버를 `강퇴`시킬 수 있다. (단, 다른 그룹장은 강퇴 불가)
- **US-4.7**: 그룹장은 멤버의 역할을 `ADMIN`(관리자)으로 지정하거나, `MEMBER`로 강등할 수 있다.

## 3. API 명세 (요약)

- `GET /api/v1/studies`: 스터디 그룹 목록 조회
- `POST /api/v1/studies`: 스터디 그룹 생성
- `GET /api/v1/studies/{studyId}`: 스터디 그룹 상세 조회
- `PATCH /api/v1/studies/{studyId}`: 스터디 그룹 수정
- `DELETE /api/v1/studies/{studyId}`: 스터디 그룹 삭제
- `GET /api/v1/studies/{studyId}/members`: 멤버 목록 조회
- `POST /api/v1/studies/{studyId}/join`: 가입 신청
- `POST /api/v1/studies/{studyId}/manage`: 가입 신청 처리
- `DELETE /api/v1/studies/{studyId}/members/{memberId}`: 멤버 강퇴/탈퇴

## 4. 데이터 모델

- `StudyGroup` 테이블
- `StudyMember` 테이블
