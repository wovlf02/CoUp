# 17. 범용 확인 모달 (General Confirmation Modal)

## 1. 목적
사용자가 되돌릴 수 없는 중요한 작업을 수행하기 전에, 해당 작업의 내용을 명확히 인지하고 최종적으로 확인하도록 유도하여 실수로 인한 데이터 손실이나 원치 않는 변경을 방지합니다.

## 2. 주요 요소
*   **모달 헤더**: "확인" 또는 "경고" 타이틀, 닫기 버튼 (X 아이콘)
*   **경고 메시지**: 수행하려는 작업에 대한 명확하고 간결한 설명 (예: "정말로 계정을 탈퇴하시겠습니까? 모든 데이터가 삭제됩니다.")
*   **확인 버튼**: 작업을 최종적으로 승인하고 진행합니다.
*   **취소 버튼**: 작업을 취소하고 모달을 닫습니다.

## 3. 레이아웃
*   **중앙 정렬 모달**: 화면 중앙에 오버레이와 함께 모달이 표시됩니다.
*   **콘텐츠 레이아웃**: 헤더, 메시지, 버튼 그룹이 세로로 배치됩니다.
*   **버튼 그룹**: 하단 우측에 확인 버튼과 취소 버튼이 배치됩니다.
*   **반응형 디자인**: 모바일 환경에서도 모달이 화면에 최적화되어 표시됩니다.

## 4. 상호작용
*   **닫기 버튼/오버레이 클릭**: 모달이 닫힙니다.
*   **확인 버튼 클릭**: 해당 작업을 수행하는 백엔드 API를 호출하고, 성공 시 모달을 닫고 관련 UI를 업데이트합니다.
*   **취소 버튼 클릭**: 모달을 닫고 작업을 취소합니다.

## 5. 디자인 고려사항
*   **경고 색상**: 확인 버튼은 작업의 중요도에 따라 경고 색상(Accent Color Red 계열)을 사용할 수 있습니다. (예: 삭제 작업)
*   **명확한 메시지**: 메시지는 사용자가 어떤 결과를 초래할지 명확하게 이해할 수 있도록 구체적이고 간결해야 합니다.
*   **사용자 집중**: 모달이 활성화되면 배경을 어둡게 처리하여 사용자의 시선이 모달에 집중되도록 합니다.

## 6. UI 요소 상세 (Detailed UI Elements)

### 6.1. 모달 오버레이 (Modal Overlay)
*   **구성**: 화면 전체를 덮는 반투명 배경.
*   **스타일**: 배경색 Neutral Dark Color (opacity 0.5), 클릭 시 모달 닫힘.

### 6.2. 모달 컨테이너 (Modal Container)
*   **구성**: 모달 헤더, 경고 메시지, 버튼 그룹.
*   **크기**: 데스크톱: 너비 400px, 높이 자동. 모바일: 너비 90%, 높이 자동.
*   **배경**: 흰색 (#FFFFFF).
*   **스타일**: 둥근 모서리 (8px), 그림자 (shadow-xl), 중앙 정렬.
*   **패딩**: 내부 패딩 30px.

### 6.3. 모달 헤더 (Modal Header)
*   **구성**: 타이틀 (H3), 닫기 버튼 (Material Icons - close).
*   **스타일**: 타이틀 폰트 20px, 굵은 글씨. 닫기 버튼은 우측 상단에 배치.
*   **타이틀 텍스트**: "확인" 또는 "경고".

### 6.4. 경고 메시지 (Warning Message)
*   **구성**: P 태그, 작업에 대한 설명 텍스트.
*   **스타일**: 폰트 16px, Neutral Dark Color, 중앙 정렬.
*   **예시**: "정말로 [작업명] 하시겠습니까? [영향]".

### 6.5. 버튼 그룹 (Button Group)
*   **구성**: "취소" 버튼 (Secondary Outline Button), "확인" 버튼 (Primary Button 또는 Accent Color Red 계열 Primary Button).
*   **정렬**: 우측 하단 정렬, 버튼 사이에 16px 간격.
*   **크기**: 높이 48px.
