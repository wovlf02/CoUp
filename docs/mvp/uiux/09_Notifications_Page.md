# 09. 알림 목록 페이지 (Notifications Page)

## 1. 목적
사용자가 서비스 내에서 발생한 모든 알림을 한곳에서 확인하고 관리할 수 있도록 합니다.

## 2. 주요 요소
*   **상단 헤더**: CoUp 로고, 사용자 프로필 이미지/닉네임, 알림 아이콘, 검색 바 (전역 헤더)
*   **좌측 사이드바**: 주요 메뉴 (전역 사이드바)
*   **페이지 타이틀**: "알림"
*   **알림 목록**: 최신순으로 정렬된 알림 항목들
*   **알림 항목**: 
    *   **읽음/안 읽음 상태**: 시각적으로 구분 (예: 안 읽은 알림은 배경색 강조 또는 굵은 글씨)
    *   **알림 내용**: "OOO 스터디 가입이 승인되었습니다."
    *   **발생 시간**: "5분 전", "어제", "2025년 10월 27일"
    *   **관련 링크**: 알림 클릭 시 이동할 페이지 (예: 스터디 상세 페이지)
*   **모두 읽음 처리 버튼**: 모든 알림을 읽음 상태로 변경합니다.
*   **페이지네이션 또는 무한 스크롤**: 알림이 많을 경우 효율적인 로딩을 위한 기능

## 3. 레이아웃
*   **2단 레이아웃**: 좌측 전역 사이드바, 우측 알림 목록 콘텐츠 영역으로 구성됩니다.
*   **알림 목록**: 각 알림 항목이 세로로 나열되는 리스트 형태로 표시됩니다.
*   **반응형 디자인**: 모바일 환경에서는 전역 사이드바가 햄버거 메뉴로 대체되고, 알림 목록이 화면에 최적화되어 표시됩니다.

## 4. 상호작용
*   **알림 항목 클릭**: 해당 알림과 관련된 페이지로 이동하고, 해당 알림은 자동으로 읽음 상태로 변경됩니다.
*   **모두 읽음 처리 버튼 클릭**: 모든 알림을 읽음 상태로 변경하는 API를 호출합니다.
*   **스크롤**: 무한 스크롤 시 이전 알림을 추가로 로드합니다.

## 5. 디자인 고려사항
*   **시각적 구분**: 읽은 알림과 안 읽은 알림을 명확하게 구분하여 사용자가 중요한 알림을 놓치지 않도록 합니다.
*   **간결한 내용**: 알림 내용은 핵심 정보를 간결하게 전달하고, 상세 내용은 링크를 통해 이동하도록 유도합니다.
*   **시간 표시**: 알림 발생 시간을 사용자 친화적인 형태로 표시합니다 (예: "방금 전", "1시간 전").
*   **빈 상태 처리**: 알림이 없을 경우, "새로운 알림이 없습니다."와 같은 안내 메시지를 표시합니다.
*   **성능**: 알림 목록이 많아질 경우를 대비하여 효율적인 데이터 로딩 및 렌더링을 고려합니다.

## 6. UI 요소 상세 (Detailed UI Elements)

### 6.1. 상단 헤더 (Global Header) 및 좌측 사이드바 (Left Sidebar)
*   메인 대시보드 페이지와 동일한 구성 및 스타일을 따릅니다.

### 6.2. 페이지 타이틀 (Page Title)
*   **텍스트**: "알림".
*   **스타일**: H2 태그, Noto Sans KR, 폰트 크기 28px, Neutral Dark Color, 좌측 정렬.

### 6.3. 모두 읽음 처리 버튼 (Mark All As Read Button)
*   **구성**: "모두 읽음" 텍스트.
*   **스타일**: Secondary Button 또는 Text Button, 폰트 14px, Primary Color 텍스트.
*   **위치**: 알림 목록 상단 우측.
*   **상태**: 읽지 않은 알림이 없을 경우 비활성화.

### 6.4. 알림 목록 (Notification List)
*   **구성**: 개별 알림 항목들의 리스트.
*   **레이아웃**: 각 알림 항목이 세로로 나열되는 리스트 형태.
*   **스타일**: 배경 흰색, 둥근 모서리 (8px), 가벼운 그림자 (shadow-sm), 패딩 20px.

### 6.5. 알림 항목 (Notification Item)
*   **구성**: 알림 유형 아이콘, 알림 내용, 발생 시간.
*   **스타일**:
    *   **안 읽은 알림**: 배경 Neutral Light Gray (#F0F0F0), 굵은 글씨 (알림 내용), 좌측에 Primary Color 세로 바.
    *   **읽은 알림**: 배경 흰색, 일반 글씨 (알림 내용).
*   **아이콘**: Material Icons, 알림 유형에 따라 다름 (예: 스터디 가입 승인 - check_circle, 새 공지 - info, 새 메시지 - chat).
*   **알림 내용**: P 태그, 폰트 14px, Neutral Dark Color. 관련 링크는 Secondary Color로 강조.
*   **발생 시간**: 폰트 12px, Neutral Gray Color, 우측 정렬.
*   **상태**: Hover 시 배경색 미묘하게 변화, 클릭 시 해당 링크로 이동 및 읽음 처리.

### 6.6. 빈 상태 (Empty State)
*   **구성**: 아이콘 (Material Icons - notifications_off), "새로운 알림이 없습니다." 텍스트.
*   **스타일**: 중앙 정렬, Neutral Gray Color.
