# 13. 스터디 상세 페이지 (Study Detail - Settings Page)

## 1. 목적
스터디 그룹의 그룹장 또는 관리자가 그룹의 기본 정보, 멤버 목록, 가입 요청 등을 관리하고, 그룹을 운영하는 데 필요한 설정을 변경할 수 있도록 합니다.

## 2. 주요 요소
*   **상단 헤더**: CoUp 로고, 사용자 프로필 이미지/닉네임, 알림 아이콘, 검색 바 (전역 헤더)
*   **좌측 사이드바**: 주요 메뉴 (전역 사이드바)
*   **스터디 헤더**: 스터디명, 스터디 소개 요약 등 (Overview 페이지와 동일)
*   **탭 네비게이션**: 스터디 내의 주요 기능(개요, 채팅, 공지, 파일, 캘린더, 할 일, 설정)으로 이동하는 탭 메뉴
*   **설정 메뉴**: 그룹 정보 수정, 멤버 관리, 가입 요청 관리, 그룹 삭제 등 서브 메뉴

### 2.1. 그룹 정보 수정 섹션 (그룹장/관리자)
*   **스터디 이름**: 수정 가능한 입력 필드
*   **스터디 소개**: 수정 가능한 텍스트 에어리어
*   **카테고리**: 수정 가능한 선택 필드
*   **모집 인원**: 수정 가능한 숫자 입력 필드
*   **공개 여부**: 수정 가능한 라디오 버튼 또는 토글 스위치
*   **저장 버튼**: 변경된 그룹 정보를 저장합니다.

### 2.2. 멤버 관리 섹션 (그룹장/관리자)
*   **멤버 목록**: 현재 스터디에 참여 중인 멤버들의 리스트
*   **멤버 항목**: 
    *   **프로필 이미지/닉네임**: 멤버 정보
    *   **역할**: 현재 멤버의 역할(그룹장, 관리자, 멤버)
    *   **역할 변경 버튼**: 그룹장/관리자가 멤버의 역할을 변경할 수 있는 기능
    *   **강퇴 버튼**: 그룹장/관리자가 멤버를 스터디에서 강퇴할 수 있는 기능

### 2.3. 가입 요청 관리 섹션 (그룹장/관리자)
*   **가입 요청 목록**: 스터디 가입을 요청한 사용자들의 리스트
*   **요청 항목**: 
    *   **프로필 이미지/닉네임**: 요청 사용자 정보
    *   **요청 메시지**: 사용자가 가입 시 작성한 메시지
    *   **승인 버튼**: 가입 요청을 승인합니다.
    *   **거절 버튼**: 가입 요청을 거절합니다.

### 2.4. 그룹 삭제 섹션 (그룹장)
*   **그룹 삭제 버튼**: 스터디 그룹을 삭제합니다.

## 3. 레이아웃
*   **2단 레이아웃**: 좌측 전역 사이드바, 우측 스터디 상세 콘텐츠 영역으로 구성됩니다.
*   **설정 콘텐츠 영역**: 좌측에 설정 서브 메뉴, 우측에 선택된 서브 메뉴에 해당하는 상세 설정 폼 또는 목록이 표시됩니다.
*   **반응형 디자인**: 모바일 환경에서는 설정 서브 메뉴가 상단 탭 또는 드롭다운으로 변경될 수 있습니다.

## 4. 상호작용
*   **설정 서브 메뉴 클릭**: 해당 섹션의 콘텐츠로 전환됩니다.
*   **그룹 정보 저장**: 유효성 검사 후 백엔드 API를 호출하여 정보를 업데이트합니다.
*   **멤버 역할 변경/강퇴**: 확인 팝업 후 백엔드 API를 호출하여 처리합니다.
*   **가입 요청 승인/거절**: 백엔드 API를 호출하여 처리하고, UI를 업데이트합니다.
*   **그룹 삭제**: 경고 메시지와 함께 최종 확인 후 그룹을 삭제합니다.

## 5. 디자인 고려사항
*   **권한에 따른 UI**: 각 섹션과 버튼은 사용자 역할(그룹장, 관리자, 일반 멤버)에 따라 다르게 표시되거나 비활성화되어야 합니다.
*   **명확한 피드백**: 정보 저장, 멤버 변경, 요청 처리 등 모든 상호작용에 대해 성공/실패 메시지를 명확하게 제공합니다.
*   **경고 및 확인**: 그룹 삭제, 멤버 강퇴 등 되돌릴 수 없는 작업에는 반드시 경고 메시지와 함께 사용자의 최종 확인을 거치도록 합니다.
*   **사용자 친화적인 폼**: 그룹 정보 수정 폼은 직관적이고 사용하기 편리하게 디자인합니다.

## 6. UI 요소 상세 (Detailed UI Elements)

### 6.1. 상단 헤더 (Global Header) 및 좌측 사이드바 (Left Sidebar)
*   메인 대시보드 페이지와 동일한 구성 및 스타일을 따릅니다.

### 6.2. 스터디 헤더 (Study Header) 및 탭 네비게이션 (Tab Navigation)
*   스터디 개요 페이지와 동일한 구성 및 스타일을 따릅니다.

### 6.3. 설정 콘텐츠 영역 (Settings Content Area)
*   **구성**: 좌측 설정 서브 메뉴, 우측 상세 설정 폼/목록.
*   **레이아웃**: 2단 레이아웃 (데스크톱), 1단 레이아웃 (모바일).
*   **스타일**: 배경 흰색 (#FFFFFF), 둥근 모서리 (8px), 가벼운 그림자 (shadow-sm), 패딩 20px.

### 6.4. 설정 서브 메뉴 (Settings Sub-Menu)
*   **구성**: "그룹 정보 수정", "멤버 관리", "가입 요청 관리", "그룹 삭제" 링크 리스트.
*   **스타일**: 세로 리스트, 선택된 메뉴는 Primary Color 배경 또는 텍스트 강조.
*   **권한**: 그룹장/관리자에게만 모든 메뉴 표시. 일반 멤버는 제한된 메뉴만 표시.

### 6.5. 그룹 정보 수정 폼 (Group Info Edit Form)
*   **구성**: 스터디 이름, 소개, 카테고리, 스터디 규칙, 모집 인원, 공개 여부 입력 필드, 저장 버튼.
*   **스타일**: 스터디 생성 페이지의 폼과 유사한 스타일. 각 필드에 현재 정보가 미리 채워져 있음.

### 6.6. 멤버 관리 목록 (Member Management List)
*   **구성**: 멤버 프로필 이미지, 닉네임, 현재 역할, 역할 변경 드롭다운, 강퇴 버튼.
*   **멤버 항목**: 
    *   **프로필 이미지**: 32x32px 원형.
    *   **닉네임**: 폰트 14px, Neutral Dark Color.
    *   **역할 변경 드롭다운**: "그룹장", "관리자", "멤버" 선택 가능. 그룹장은 자신의 역할을 변경할 수 없음.
    *   **강퇴 버튼**: Material Icons (person_remove), 경고 색상 (Accent Color Red 계열).
*   **권한**: 그룹장/관리자에게만 역할 변경 및 강퇴 버튼 표시. 그룹장은 다른 그룹장을 강퇴할 수 없음.

### 6.7. 가입 요청 목록 (Join Request List)
*   **구성**: 요청자 프로필 이미지, 닉네임, 요청 메시지, 승인 버튼, 거절 버튼.
*   **요청 항목**: 
    *   **프로필 이미지**: 32x32px 원형.
    *   **닉네임**: 폰트 14px, Neutral Dark Color.
    *   **요청 메시지**: P 태그, 폰트 12px, Neutral Gray Color.
    *   **승인 버튼**: Primary Button, 작은 크기.
    *   **거절 버튼**: Secondary Outline Button, 작은 크기.
*   **빈 상태**: 가입 요청이 없을 경우 "새로운 가입 요청이 없습니다." 메시지 표시.

### 6.8. 그룹 삭제 버튼 (Delete Group Button)
*   **구성**: "그룹 삭제" 텍스트.
*   **스타일**: 경고 색상 (Accent Color Red 계열) Primary Button, 높이 48px, 너비 150px.
*   **위치**: 별도 섹션 또는 설정 폼 하단.
*   **모달 (Modal)**: 클릭 시 경고 메시지 ("정말로 스터디 그룹을 삭제하시겠습니까? 모든 데이터가 삭제됩니다."), "확인" (경고 버튼), "취소" 버튼 포함.
