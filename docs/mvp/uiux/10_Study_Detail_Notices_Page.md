# 10. 스터디 상세 페이지 (Study Detail - Notices Page)

## 1. 목적
스터디 그룹 멤버들이 그룹의 중요한 공지사항을 확인하고, 그룹장/관리자는 새로운 공지를 등록 및 관리할 수 있도록 합니다.

## 2. 주요 요소
*   **상단 헤더**: CoUp 로고, 사용자 프로필 이미지/닉네임, 알림 아이콘, 검색 바 (전역 헤더)
*   **좌측 사이드바**: 주요 메뉴 (전역 사이드바)
*   **스터디 헤더**: 스터디명, 스터디 소개 요약 등 (Overview 페이지와 동일)
*   **탭 네비게이션**: 스터디 내의 주요 기능(개요, 채팅, 공지, 파일, 캘린더, 할 일, 설정)으로 이동하는 탭 메뉴
*   **공지사항 목록**: 최신순으로 정렬된 공지사항 항목들
*   **공지사항 항목**: 
    *   **제목**: 공지사항의 제목
    *   **작성자**: 공지사항을 작성한 그룹장/관리자의 닉네임
    *   **작성일**: 공지사항이 작성된 날짜
    *   **조회수**: (선택 사항) 공지사항 조회수
*   **새 공지 작성 버튼**: 그룹장/관리자에게만 표시되는 공지 작성 버튼
*   **공지 상세 보기**: 공지사항 항목 클릭 시 모달 또는 별도 페이지로 상세 내용 표시

## 3. 레이아웃
*   **2단 레이아웃**: 좌측 전역 사이드바, 우측 스터디 상세 콘텐츠 영역으로 구성됩니다.
*   **공지사항 목록**: 스터디 상세 콘텐츠 영역 내에서 공지사항 목록이 리스트 형태로 표시됩니다.
*   **반응형 디자인**: 모바일 환경에서는 공지사항 목록이 화면에 최적화되어 표시됩니다.

## 4. 상호작용
*   **공지사항 항목 클릭**: 해당 공지사항의 상세 내용을 모달 또는 별도 페이지에서 확인합니다.
*   **새 공지 작성 버튼 클릭**: 공지 작성 폼이 있는 모달 또는 페이지로 이동합니다.
*   **공지 수정/삭제**: 그룹장/관리자는 각 공지사항 옆에 수정/삭제 버튼을 통해 관리할 수 있습니다.

## 5. 디자인 고려사항
*   **중요도 강조**: 중요한 공지사항은 상단에 고정하거나, 배경색을 다르게 하여 시각적으로 강조할 수 있습니다.
*   **간결한 목록**: 목록에서는 제목, 작성자, 작성일 등 핵심 정보만 표시하고, 상세 내용은 별도로 제공합니다.
*   **권한에 따른 UI**: 그룹장/관리자에게만 공지 작성, 수정, 삭제 버튼이 보이도록 처리하여 혼란을 방지합니다.
*   **빈 상태 처리**: 공지사항이 없을 경우, "아직 등록된 공지사항이 없습니다."와 같은 안내 메시지를 표시합니다.

## 6. UI 요소 상세 (Detailed UI Elements)

### 6.1. 상단 헤더 (Global Header) 및 좌측 사이드바 (Left Sidebar)
*   메인 대시보드 페이지와 동일한 구성 및 스타일을 따릅니다.

### 6.2. 스터디 헤더 (Study Header) 및 탭 네비게이션 (Tab Navigation)
*   스터디 개요 페이지와 동일한 구성 및 스타일을 따릅니다.

### 6.3. 공지사항 목록 영역 (Notices List Area)
*   **구성**: 페이지 타이틀, 새 공지 작성 버튼, 공지사항 목록, 페이지네이션 (선택 사항).
*   **레이아웃**: 스터디 상세 콘텐츠 영역 내에서 상단에 타이틀과 버튼, 하단에 목록.
*   **스타일**: 배경 흰색 (#FFFFFF), 둥근 모서리 (8px), 가벼운 그림자 (shadow-sm), 패딩 20px.

### 6.4. 페이지 타이틀 (Page Title)
*   **텍스트**: "공지사항".
*   **스타일**: H3 태그, Noto Sans KR, 폰트 크기 24px, Neutral Dark Color, 좌측 정렬.

### 6.5. 새 공지 작성 버튼 (Create New Notice Button)
*   **구성**: "새 공지 작성" 텍스트, 아이콘 (Material Icons - add).
*   **스타일**: Primary Button, 높이 40px, 너비 150px.
*   **위치**: 페이지 타이틀 우측.
*   **권한**: 그룹장/관리자에게만 표시.

### 6.6. 공지사항 목록 (Notices List)
*   **구성**: 개별 공지사항 항목들의 리스트.
*   **레이아웃**: 테이블 또는 리스트 형태로 표시.
*   **스타일**: 각 항목은 하단 테두리 (Neutral Light Gray)로 구분.

### 6.7. 공지사항 항목 (Notice Item)
*   **구성**: 제목, 작성자, 작성일, (선택적으로 조회수), 수정/삭제 버튼 (그룹장/관리자).
*   **제목**: 폰트 16px, 굵은 글씨, Neutral Dark Color. Hover 시 밑줄 효과.
*   **작성자/작성일/조회수**: 폰트 12px, Neutral Gray Color.
*   **수정/삭제 버튼**: Material Icons (edit, delete), 작은 크기, Secondary Outline Button 스타일. Hover 시 색상 변화.
*   **상태**: 클릭 시 공지 상세 모달/페이지로 이동.

### 6.8. 공지 상세 모달 (Notice Detail Modal)
*   **구성**: 모달 헤더 (제목, 닫기 버튼), 작성자/작성일 정보, 공지 내용, (그룹장/관리자에게) 수정/삭제 버튼.
*   **스타일**: 배경 흰색, 둥근 모서리, 그림자. 내용 영역은 스크롤 가능.
*   **공지 내용**: Markdown 렌더링 지원.
