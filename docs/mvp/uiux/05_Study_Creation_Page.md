# 05. 스터디 생성 페이지 (Study Creation Page)

## 1. 목적
사용자가 자신의 학습 목표와 필요에 맞는 새로운 스터디 그룹을 쉽고 체계적으로 생성할 수 있도록 돕습니다.

## 2. 주요 요소
*   **페이지 타이틀**: "새 스터디 만들기"
*   **입력 폼**: 
    *   **스터디 이름**: (필수) 텍스트 입력 필드
    *   **스터디 소개**: (필수) 텍스트 에어리어 (Markdown 지원 예정)
    *   **스터디 목표**: (선택) 텍스트 입력 필드
    *   **카테고리**: (필수) 드롭다운 또는 태그 선택 (예: 프로그래밍, 어학, 자격증)
    *   **모집 인원**: (선택) 숫자 입력 필드 또는 슬라이더 (기본값 10명)
    *   **공개 여부**: (필수) 라디오 버튼 또는 토글 스위치 (공개/비공개)
*   **생성 버튼**: 입력된 정보를 바탕으로 스터디를 생성합니다.
*   **취소 버튼**: 스터디 생성을 취소하고 이전 페이지로 돌아갑니다.

## 3. 레이아웃
*   **단일 컬럼 폼**: 모든 입력 필드가 세로로 나열된 단일 컬럼 형태로 구성되어 사용자가 순서대로 정보를 입력하기 쉽습니다.
*   **단계별 진행 (선택 사항)**: 입력 필드가 많아질 경우, 여러 단계로 나누어 진행 상황을 표시하는 스텝퍼(Stepper) UI를 고려할 수 있습니다. (MVP에서는 단일 폼으로 시작)
*   **반응형 디자인**: 모바일 환경에서도 입력 필드와 버튼이 적절한 크기로 조절되어 사용하기 편리합니다.

## 4. 상호작용
*   **입력 필드 유효성 검사**: 각 필드에 대한 유효성 검사(필수 항목, 길이 제한 등)를 실시간으로 수행하고, 오류 발생 시 명확한 피드백을 제공합니다.
*   **카테고리 선택**: 드롭다운 또는 태그 선택 시 시각적인 피드백을 제공합니다.
*   **생성 버튼 클릭**: 모든 유효성 검사를 통과하면 백엔드 API를 호출하여 스터디를 생성하고, 성공 시 생성된 스터디의 상세 페이지로 이동합니다.
*   **취소 버튼 클릭**: 확인 팝업 후 이전 페이지로 돌아갑니다.

## 5. 디자인 고려사항
*   **명확한 안내**: 각 입력 필드에 대한 플레이스홀더 텍스트나 툴팁을 제공하여 사용자가 어떤 정보를 입력해야 하는지 쉽게 이해할 수 있도록 돕습니다.
*   **시각적 계층**: 필수 입력 필드는 별도로 표시(예: * 표시)하여 사용자가 놓치지 않도록 합니다.
*   **버튼 스타일**: Primary Color를 사용하여 "생성" 버튼을 강조하고, "취소" 버튼은 보조적인 스타일을 적용합니다.
*   **피드백**: 스터디 생성 중에는 로딩 스피너를 표시하고, 성공/실패 시 토스트 메시지 등의 알림을 제공합니다.

## 6. UI 요소 상세 (Detailed UI Elements)

### 6.1. 상단 헤더 (Global Header) 및 좌측 사이드바 (Left Sidebar)
*   메인 대시보드 페이지와 동일한 구성 및 스타일을 따릅니다.

### 6.2. 페이지 타이틀 (Page Title)
*   **텍스트**: "새 스터디 만들기".
*   **스타일**: H2 태그, Noto Sans KR, 폰트 크기 28px, Neutral Dark Color, 좌측 정렬.

### 6.3. 스터디 생성 폼 (Study Creation Form)
*   **구성**: 각 입력 필드와 라벨, 도움말 텍스트.
*   **레이아웃**: 단일 컬럼, 각 필드 사이에 20px 간격.
*   **스타일**: 배경 흰색, 둥근 모서리 (8px), 가벼운 그림자 (shadow-sm), 패딩 30px.

### 6.4. 입력 필드 (Input Field)
*   **구성**: 라벨, 입력 요소 (Input, Textarea, Select, Toggle), 도움말/오류 메시지.
*   **라벨**: 폰트 14px, 굵은 글씨, Neutral Dark Color.
*   **Input/Textarea**: 
    *   **스타일**: 테두리 Neutral Light Gray, 둥근 모서리 (4px), 패딩 10px.
    *   **상태**: Focus 시 Primary Color 테두리, Error 시 Accent Color 테두리 및 텍스트.
*   **Textarea (스터디 소개)**: 최소 높이 120px, Markdown 미리보기/가이드 버튼 (Post-MVP).
*   **카테고리 (Dropdown/Tag Select)**:
    *   **구성**: 드롭다운 메뉴 또는 선택 가능한 태그 목록.
    *   **스타일**: 선택된 태그는 Primary Color 배경, 흰색 텍스트.
*   **모집 인원 (Number Input/Slider)**:
    *   **구성**: 숫자 입력 필드 또는 슬라이더 컴포넌트.
    *   **플레이스홀더**: "최대 10명 (선택 사항)".
*   **공개 여부 (Toggle Switch)**:
    *   **구성**: "공개" / "비공개" 라벨과 토글 스위치.
    *   **스타일**: 활성화 시 Primary Color 배경.

### 6.5. 버튼 그룹 (Button Group)
*   **구성**: "생성" 버튼 (Primary Button), "취소" 버튼 (Secondary Outline Button).
*   **정렬**: 우측 정렬, 버튼 사이에 16px 간격.
*   **크기**: 높이 48px.
