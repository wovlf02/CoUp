# 15. 일정 추가/수정 모달 (Event Add/Edit Modal)

## 1. 목적
스터디 그룹의 그룹장 또는 관리자가 새로운 스터디 일정을 추가하거나 기존 일정을 수정할 수 있도록 합니다.

## 2. 주요 요소
*   **모달 헤더**: "새 일정 추가" 또는 "일정 수정" 타이틀, 닫기 버튼 (X 아이콘)
*   **제목 입력 필드**: 일정의 제목을 입력합니다.
*   **설명 입력 필드**: 일정에 대한 상세 설명을 입력합니다.
*   **시작일/종료일 선택**: 달력 컴포넌트를 통해 시작일과 종료일을 선택합니다.
*   **시작 시간/종료 시간 선택**: 시간 선택 컴포넌트를 통해 시작 시간과 종료 시간을 선택합니다.
*   **추가/수정 버튼**: 입력된 내용을 저장하고 모달을 닫습니다.
*   **취소 버튼**: 추가/수정을 취소하고 모달을 닫습니다.

## 3. 레이아웃
*   **중앙 정렬 모달**: 화면 중앙에 오버레이와 함께 모달이 표시됩니다.
*   **폼 레이아웃**: 제목, 설명, 날짜/시간 선택 필드가 세로로 배치됩니다.
*   **버튼 그룹**: 하단 우측에 추가/수정 버튼과 취소 버튼이 배치됩니다.
*   **반응형 디자인**: 모바일 환경에서는 모달이 전체 화면을 차지하거나, 내용이 스크롤 가능하도록 조정됩니다.

## 4. 상호작용
*   **닫기 버튼/오버레이 클릭**: 모달이 닫힙니다.
*   **제목/설명 입력**: 실시간으로 입력 내용을 반영합니다.
*   **날짜/시간 선택**: 달력/시간 선택 컴포넌트를 통해 직관적으로 값을 설정합니다.
*   **유효성 검사**: 필수 항목이 비어있거나 날짜/시간 순서가 잘못되었을 경우 오류 메시지를 표시합니다.
*   **추가/수정 버튼 클릭**: 백엔드 API를 호출하여 일정을 저장/업데이트하고, 성공 시 모달을 닫고 캘린더를 새로고침합니다.
*   **취소 버튼 클릭**: 모달을 닫고 변경사항을 폐기합니다.

## 5. 디자인 고려사항
*   **달력/시간 선택 컴포넌트**: 사용자 친화적이고 직관적인 컴포넌트를 사용하여 날짜/시간 선택의 편의성을 높입니다.
*   **버튼 스타일**: "추가" 또는 "수정" 버튼은 Primary Color로 강조하고, "취소" 버튼은 보조적인 스타일을 적용합니다.
*   **피드백**: 저장 중에는 로딩 스피너를 표시하고, 성공/실패 시 토스트 메시지 등의 알림을 제공합니다.

## 6. UI 요소 상세 (Detailed UI Elements)

### 6.1. 모달 오버레이 (Modal Overlay)
*   **구성**: 화면 전체를 덮는 반투명 배경.
*   **스타일**: 배경색 Neutral Dark Color (opacity 0.5), 클릭 시 모달 닫힘.

### 6.2. 모달 컨테이너 (Modal Container)
*   **구성**: 모달 헤더, 폼 요소, 버튼 그룹.
*   **크기**: 데스크톱: 너비 500px, 높이 자동. 모바일: 너비 90%, 높이 자동.
*   **배경**: 흰색 (#FFFFFF).
*   **스타일**: 둥근 모서리 (8px), 그림자 (shadow-xl), 중앙 정렬.
*   **패딩**: 내부 패딩 30px.

### 6.3. 모달 헤더 (Modal Header)
*   **구성**: 타이틀 (H3), 닫기 버튼 (Material Icons - close).
*   **스타일**: 타이틀 폰트 20px, 굵은 글씨. 닫기 버튼은 우측 상단에 배치.

### 6.4. 제목 입력 필드 (Title Input Field)
*   **구성**: 라벨 ("제목"), Input 필드.
*   **스타일**: 일반 Input Field와 동일. 높이 48px.
*   **플레이스홀더**: "일정 제목을 입력하세요.".

### 6.5. 설명 입력 필드 (Description Input Field)
*   **구성**: 라벨 ("설명"), Textarea 필드.
*   **스타일**: 일반 Textarea와 동일. 최소 높이 100px.
*   **플레이스홀더**: "일정에 대한 상세 설명을 입력하세요.".

### 6.6. 날짜/시간 선택 필드 (Date/Time Picker Fields)
*   **구성**: 라벨 ("시작일", "종료일", "시작 시간", "종료 시간"), 달력 컴포넌트, 시간 선택 컴포넌트.
*   **스타일**: 각 필드는 Input Field 형태로 표시되며, 클릭 시 달력/시간 선택 UI가 팝업됩니다.
*   **유효성 검사**: 종료일/시간이 시작일/시간보다 이전일 경우 오류 메시지 표시.

### 6.7. 버튼 그룹 (Button Group)
*   **구성**: "취소" 버튼 (Secondary Outline Button), "추가" 또는 "수정" 버튼 (Primary Button).
*   **정렬**: 우측 하단 정렬, 버튼 사이에 16px 간격.
*   **크기**: 높이 48px.
