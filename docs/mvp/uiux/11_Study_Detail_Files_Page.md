# 11. 스터디 상세 페이지 (Study Detail - Files Page)

## 1. 목적
스터디 그룹 멤버들이 학습 자료를 업로드하고 공유하며, 필요한 파일을 쉽게 다운로드할 수 있도록 합니다.

## 2. 주요 요소
*   **상단 헤더**: CoUp 로고, 사용자 프로필 이미지/닉네임, 알림 아이콘, 검색 바 (전역 헤더)
*   **좌측 사이드바**: 주요 메뉴 (전역 사이드바)
*   **스터디 헤더**: 스터디명, 스터디 소개 요약 등 (Overview 페이지와 동일)
*   **탭 네비게이션**: 스터디 내의 주요 기능(개요, 채팅, 공지, 파일, 캘린더, 할 일, 화상 스터디, 설정)으로 이동하는 탭 메뉴
*   **파일 목록**: 업로드된 파일들을 리스트 또는 그리드 형태로 표시
*   **파일 항목**: 
    *   **파일 아이콘**: 파일 종류(PDF, 이미지, ZIP 등)에 따른 아이콘
    *   **파일 이름**: 업로드된 파일의 이름
    *   **업로더**: 파일을 업로드한 사용자 닉네임
    *   **업로드 날짜**: 파일이 업로드된 날짜
    *   **파일 크기**: 파일의 용량
    *   **다운로드 버튼**: 파일 다운로드 기능
    *   **삭제 버튼**: (업로더 또는 그룹장/관리자에게만 표시) 파일 삭제 기능
*   **파일 업로드 영역**: 드래그 앤 드롭 또는 파일 선택 버튼을 통한 파일 업로드 기능

## 3. 레이아웃
*   **2단 레이아웃**: 좌측 전역 사이드바, 우측 스터디 상세 콘텐츠 영역으로 구성됩니다.
*   **파일 목록**: 스터디 상세 콘텐츠 영역 내에서 파일 목록이 표시되며, 상단에 파일 업로드 영역이 위치합니다.
*   **반응형 디자인**: 모바일 환경에서는 파일 목록과 업로드 영역이 화면에 최적화되어 표시됩니다.

## 4. 상호작용
*   **파일 업로드**: 드래그 앤 드롭 또는 버튼 클릭으로 파일을 선택하여 업로드합니다. 업로드 진행 상황을 표시합니다.
*   **파일 다운로드**: 파일 항목 옆의 다운로드 버튼 클릭 시 파일을 다운로드합니다.
*   **파일 삭제**: (권한이 있는 경우) 파일 항목 옆의 삭제 버튼 클릭 시 확인 팝업 후 파일을 삭제합니다.
*   **파일 검색/필터링**: (Post-MVP) 파일 이름, 업로더, 파일 종류 등으로 검색하거나 필터링할 수 있습니다.

## 5. 디자인 고려사항
*   **직관적인 업로드**: 드래그 앤 드롭 영역을 명확하게 표시하고, 파일 선택 버튼을 눈에 띄게 배치합니다.
*   **업로드 피드백**: 파일 업로드 중에는 진행률 바 또는 로딩 스피너를 표시하고, 성공/실패 시 알림을 제공합니다.
*   **파일 아이콘**: 파일 종류별로 다른 아이콘을 사용하여 시각적으로 쉽게 구분할 수 있도록 합니다.
*   **권한 관리**: 파일 삭제 버튼은 해당 파일을 업로드한 사용자 또는 그룹장/관리자에게만 보이도록 처리합니다.
*   **빈 상태 처리**: 공유된 파일이 없을 경우, "아직 공유된 파일이 없습니다. 첫 파일을 업로드해보세요!"와 같은 안내 메시지를 표시합니다.

## 6. UI 요소 상세 (Detailed UI Elements)

### 6.1. 상단 헤더 (Global Header) 및 좌측 사이드바 (Left Sidebar)
*   메인 대시보드 페이지와 동일한 구성 및 스타일을 따릅니다.

### 6.2. 스터디 헤더 (Study Header) 및 탭 네비게이션 (Tab Navigation)
*   스터디 개요 페이지와 동일한 구성 및 스타일을 따릅니다.

### 6.3. 파일 공유 영역 (File Sharing Area)
*   **구성**: 페이지 타이틀, 파일 업로드 영역, 파일 목록.
*   **레이아웃**: 스터디 상세 콘텐츠 영역 내에서 상단에 타이틀과 업로드 영역, 하단에 파일 목록.
*   **스타일**: 배경 흰색 (#FFFFFF), 둥근 모서리 (8px), 가벼운 그림자 (shadow-sm), 패딩 20px.

### 6.4. 페이지 타이틀 (Page Title)
*   **텍스트**: "파일 공유".
*   **스타일**: H3 태그, Noto Sans KR, 폰트 크기 24px, Neutral Dark Color, 좌측 정렬.

### 6.5. 파일 업로드 영역 (File Upload Area)
*   **구성**: 드래그 앤 드롭 영역, "파일 선택" 버튼, 업로드 가이드 텍스트 (예: "여기에 파일을 드래그하거나 클릭하여 업로드하세요.").
*   **스타일**: 점선 테두리 (Neutral Light Gray), 둥근 모서리 (8px), 배경 Neutral Light Gray Light, 높이 150px.
*   **상태**: 파일 드래그 시 테두리 색상 Primary Color로 변경.

### 6.6. 파일 목록 (File List)
*   **구성**: 개별 파일 항목들의 리스트.
*   **레이아웃**: 테이블 또는 리스트 형태로 표시.
*   **스타일**: 각 항목은 하단 테두리 (Neutral Light Gray)로 구분.

### 6.7. 파일 항목 (File Item)
*   **구성**: 파일 아이콘, 파일 이름, 업로더, 업로드 날짜, 파일 크기, 다운로드 버튼, 삭제 버튼.
*   **파일 아이콘**: Material Icons, 파일 확장자에 따라 다름 (예: PDF - picture_as_pdf, 이미지 - image, ZIP - folder_zip).
*   **파일 이름**: 폰트 16px, Neutral Dark Color. Hover 시 밑줄 효과.
*   **업로더/업로드 날짜/파일 크기**: 폰트 12px, Neutral Gray Color.
*   **다운로드 버튼**: Material Icons (download), 작은 크기, Secondary Outline Button 스타일. Hover 시 색상 변화.
*   **삭제 버튼**: Material Icons (delete), 작은 크기, Secondary Outline Button 스타일 (경고 색상). Hover 시 색상 변화.
*   **권한**: 삭제 버튼은 업로더 또는 그룹장/관리자에게만 표시.
