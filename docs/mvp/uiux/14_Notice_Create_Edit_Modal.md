# 14. 공지사항 작성/수정 모달 (Notice Create/Edit Modal)

## 1. 목적
스터디 그룹의 그룹장 또는 관리자가 새로운 공지사항을 작성하거나 기존 공지사항의 내용을 수정할 수 있도록 합니다.

## 2. 주요 요소
*   **모달 헤더**: "새 공지 작성" 또는 "공지 수정" 타이틀, 닫기 버튼 (X 아이콘)
*   **제목 입력 필드**: 공지사항의 제목을 입력합니다.
*   **내용 입력 필드**: 공지사항의 상세 내용을 입력합니다. (Markdown 에디터)
*   **작성/수정 버튼**: 입력된 내용을 저장하고 모달을 닫습니다.
*   **취소 버튼**: 작성/수정을 취소하고 모달을 닫습니다.

## 3. 레이아웃
*   **중앙 정렬 모달**: 화면 중앙에 오버레이와 함께 모달이 표시됩니다.
*   **폼 레이아웃**: 제목 입력 필드, 내용 입력 필드가 세로로 배치됩니다.
*   **버튼 그룹**: 하단 우측에 작성/수정 버튼과 취소 버튼이 배치됩니다.
*   **반응형 디자인**: 모바일 환경에서는 모달이 전체 화면을 차지하거나, 내용이 스크롤 가능하도록 조정됩니다.

## 4. 상호작용
*   **닫기 버튼/오버레이 클릭**: 모달이 닫힙니다.
*   **제목/내용 입력**: 실시간으로 입력 내용을 반영합니다.
*   **유효성 검사**: 제목 또는 내용이 비어있을 경우 작성/수정 버튼을 비활성화하거나 오류 메시지를 표시합니다.
*   **작성/수정 버튼 클릭**: 백엔드 API를 호출하여 공지사항을 저장/업데이트하고, 성공 시 모달을 닫고 공지사항 목록을 새로고침합니다.
*   **취소 버튼 클릭**: 모달을 닫고 변경사항을 폐기합니다.

## 5. 디자인 고려사항
*   **Markdown 에디터**: 사용자 친화적인 Markdown 에디터 (예: 미리보기, 툴바)를 제공하여 내용 작성을 돕습니다.
*   **버튼 스타일**: "작성" 또는 "수정" 버튼은 Primary Color로 강조하고, "취소" 버튼은 보조적인 스타일을 적용합니다.
*   **피드백**: 저장 중에는 로딩 스피너를 표시하고, 성공/실패 시 토스트 메시지 등의 알림을 제공합니다.

## 6. UI 요소 상세 (Detailed UI Elements)

### 6.1. 모달 오버레이 (Modal Overlay)
*   **구성**: 화면 전체를 덮는 반투명 배경.
*   **스타일**: 배경색 Neutral Dark Color (opacity 0.5), 클릭 시 모달 닫힘.

### 6.2. 모달 컨테이너 (Modal Container)
*   **구성**: 모달 헤더, 폼 요소, 버튼 그룹.
*   **크기**: 데스크톱: 너비 600px, 높이 자동. 모바일: 너비 90%, 높이 자동.
*   **배경**: 흰색 (#FFFFFF).
*   **스타일**: 둥근 모서리 (8px), 그림자 (shadow-xl), 중앙 정렬.
*   **패딩**: 내부 패딩 30px.

### 6.3. 모달 헤더 (Modal Header)
*   **구성**: 타이틀 (H3), 닫기 버튼 (Material Icons - close).
*   **스타일**: 타이틀 폰트 20px, 굵은 글씨. 닫기 버튼은 우측 상단에 배치.

### 6.4. 제목 입력 필드 (Title Input Field)
*   **구성**: 라벨 ("제목"), Input 필드.
*   **스타일**: 일반 Input Field와 동일. 높이 48px.
*   **플레이스홀더**: "공지사항 제목을 입력하세요.".

### 6.5. 내용 입력 필드 (Content Input Field - Markdown Editor)
*   **구성**: 라벨 ("내용"), Textarea 또는 Markdown 에디터 컴포넌트.
*   **스타일**: 최소 높이 200px. Markdown 에디터 툴바 포함 (굵게, 기울임, 링크, 목록 등).
*   **플레이스홀더**: "공지사항 내용을 Markdown 형식으로 입력하세요.".

### 6.6. 버튼 그룹 (Button Group)
*   **구성**: "취소" 버튼 (Secondary Outline Button), "작성" 또는 "수정" 버튼 (Primary Button).
*   **정렬**: 우측 하단 정렬, 버튼 사이에 16px 간격.
*   **크기**: 높이 48px.
