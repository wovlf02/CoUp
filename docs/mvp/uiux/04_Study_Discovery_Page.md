# 04. 스터디 탐색/목록 페이지 (Study Discovery/List Page)

## 1. 목적
사용자가 자신의 학습 목표와 관심사에 맞는 스터디 그룹을 쉽게 찾고 탐색할 수 있도록 합니다.

## 2. 주요 요소
*   **상단 헤더**: CoUp 로고, 사용자 프로필 이미지/닉네임, 알림 아이콘, 검색 바 (메인 대시보드와 동일)
*   **좌측 사이드바**: 주요 메뉴 (메인 대시보드와 동일)
*   **검색 필터**: 카테고리(프로그래밍, 어학, 자격증 등), 스터디 상태(모집 중, 진행 중), 공개 여부(공개/비공개), 정원 등
*   **검색 결과 목록**: 필터링된 스터디 그룹을 카드 형태로 표시
*   **스터디 카드**: 스터디명, 간단한 소개, 카테고리 태그, 현재 인원/최대 정원, 그룹장 정보, 생성일 등
*   **페이지네이션 또는 무한 스크롤**: 스터디 목록이 많을 경우 효율적인 로딩을 위한 기능
*   **새 스터디 만들기 버튼**: 스터디를 직접 생성할 수 있는 CTA 버튼

## 3. 레이아웃
*   **2단 레이아웃**: 좌측에 검색 필터 영역, 우측에 스터디 목록 영역으로 구성됩니다.
*   **스터디 목록**: 그리드(Grid) 형태로 여러 스터디 카드를 배치하여 한눈에 많은 정보를 볼 수 있도록 합니다.
*   **반응형 디자인**: 모바일 환경에서는 검색 필터가 상단으로 이동하거나 토글 형태로 제공되며, 스터디 카드는 세로로 한 줄씩 배치됩니다.

## 4. 상호작용
*   **필터 선택/해제**: 필터 옵션 변경 시 실시간으로 스터디 목록이 업데이트됩니다.
*   **검색어 입력**: 검색어 입력 시 스터디명 또는 소개 내용에서 일치하는 스터디를 검색합니다.
*   **스터디 카드 클릭**: 해당 스터디의 상세 페이지로 이동합니다.
*   **새 스터디 만들기 버튼 클릭**: 스터디 생성 페이지로 이동합니다.

## 5. 디자인 고려사항
*   **직관적인 필터**: 사용자가 원하는 스터디를 빠르게 찾을 수 있도록 필터 옵션을 명확하고 직관적으로 제공합니다.
*   **스터디 카드 디자인**: 각 스터디 카드는 핵심 정보를 간결하게 보여주면서도 시각적으로 매력적이어야 합니다.
*   **로딩 피드백**: 스터디 목록 로딩 시 스켈레톤 UI 또는 로딩 스피너를 제공하여 사용자 경험을 향상시킵니다.
*   **빈 상태 처리**: 검색 결과가 없을 경우, 사용자에게 친절한 안내 메시지와 함께 다른 스터디를 탐색하거나 직접 스터디를 만들도록 유도하는 CTA를 제공합니다.

## 6. UI 요소 상세 (Detailed UI Elements)

### 6.1. 상단 헤더 (Global Header) 및 좌측 사이드바 (Left Sidebar)
*   메인 대시보드 페이지와 동일한 구성 및 스타일을 따릅니다.

### 6.2. 검색 필터 영역 (Search Filter Area)
*   **구성**: 카테고리 드롭다운, 스터디 상태 체크박스/라디오 버튼, 공개 여부 토글, 정원 범위 슬라이더.
*   **위치**: 데스크톱에서는 좌측 사이드바 옆, 모바일에서는 상단에 토글 가능한 형태로 배치.
*   **스타일**: 각 필터는 명확한 라벨과 함께 제공되며, 선택 상태를 시각적으로 명확히 보여줍니다.
*   **카테고리 드롭다운**:
    *   **구성**: "전체 카테고리" 기본값, "프로그래밍", "어학", "자격증" 등 목록.
    *   **상태**: 선택 시 드롭다운 메뉴 확장, 항목 선택 시 메뉴 닫힘 및 필터 적용.
*   **스터디 상태**:
    *   **구성**: "모집 중", "진행 중", "종료" (체크박스 또는 라디오 버튼).
*   **공개 여부**:
    *   **구성**: "공개", "비공개" (토글 스위치).
*   **정원 범위**:
    *   **구성**: 최소/최대 인원 입력 필드 또는 슬라이더.

### 6.3. 검색 결과 목록 (Search Result List)
*   **구성**: 필터링된 스터디 그룹을 담는 컨테이너.
*   **레이아웃**: 데스크톱: 3열 그리드, 태블릿: 2열 그리드, 모바일: 1열 그리드.
*   **패딩**: 각 스터디 카드 사이에 20px 간격.

### 6.4. 스터디 카드 (Study Card)
*   **구성**:
    *   **스터디 이미지/로고**: 상단 중앙, 100% 너비, 높이 120px, 둥근 상단 모서리.
    *   **스터디명**: H4 태그, Noto Sans KR, 폰트 20px, Neutral Dark Color, 굵은 글씨.
    *   **간단한 소개**: P 태그, 폰트 14px, Neutral Gray Color, 2줄까지 표시 후 "..." 처리.
    *   **카테고리 태그**: 작은 뱃지 형태 (Primary Color 배경, 흰색 텍스트), 폰트 12px.
    *   **현재 인원/최대 정원**: 아이콘 (Material Icons - people), 텍스트 (예: "5/10명").
    *   **그룹장 정보**: 프로필 이미지 (24x24px 원형), 닉네임.
    *   **생성일**: 텍스트 (예: "2025.10.28 생성").
*   **스타일**: 카드 형태, 둥근 모서리 (8px), 그림자 (shadow-md), 배경 흰색.
*   **상태**:
    *   **Hover**: 카드에 미묘한 그림자 확장 또는 배경색 변화 효과.

### 6.5. 새 스터디 만들기 버튼 (Create New Study Button)
*   **구성**: "새 스터디 만들기" 텍스트, 아이콘 (Material Icons - add).
*   **스타일**: Primary Button, 고정 위치 (예: 화면 우측 하단 Floating Action Button 형태) 또는 목록 상단.
*   **상태**:
    *   **Hover**: 배경색 진해짐.

### 6.6. 페이지네이션/무한 스크롤 (Pagination/Infinite Scroll)
*   **구성**: 페이지 번호 버튼 또는 "더 보기" 버튼.
*   **스타일**: 중앙 정렬.
*   **로딩 피드백**: 다음 페이지 로딩 시 스피너 표시.
