# 12. 스터디 상세 페이지 (Study Detail - Calendar Page)

## 1. 목적
스터디 그룹의 모든 멤버가 스터디 일정, 모임, 과제 마감일 등을 한눈에 확인하고, 그룹장/관리자는 새로운 일정을 등록 및 관리할 수 있도록 합니다.

## 2. 주요 요소
*   **상단 헤더**: CoUp 로고, 사용자 프로필 이미지/닉네임, 알림 아이콘, 검색 바 (전역 헤더)
*   **좌측 사이드바**: 주요 메뉴 (전역 사이드바)
*   **스터디 헤더**: 스터디명, 스터디 소개 요약 등 (Overview 페이지와 동일)
*   **탭 네비게이션**: 스터디 내의 주요 기능(개요, 채팅, 공지, 파일, 캘린더, 할 일, 설정)으로 이동하는 탭 메뉴
*   **캘린더 뷰**: 월별/주별/일별로 스터디 일정을 시각적으로 표시하는 캘린더 컴포넌트
*   **일정 목록**: 선택된 날짜 또는 기간의 상세 일정 목록
*   **일정 항목**: 
    *   **제목**: 일정의 제목
    *   **시간**: 시작 시간 및 종료 시간
    *   **설명**: 간단한 일정 설명
    *   **작성자**: 일정을 등록한 사용자
*   **새 일정 추가 버튼**: 그룹장/관리자에게만 표시되는 일정 추가 버튼

## 3. 레이아웃
*   **2단 레이아웃**: 좌측 전역 사이드바, 우측 스터디 상세 콘텐츠 영역으로 구성됩니다.
*   **캘린더 뷰**: 스터디 상세 콘텐츠 영역 내에서 캘린더 컴포넌트가 대부분을 차지하며, 하단 또는 우측에 상세 일정 목록이 표시될 수 있습니다.
*   **반응형 디자인**: 모바일 환경에서는 캘린더 뷰가 스크롤 가능한 형태로 제공되거나, 리스트 뷰로 전환될 수 있습니다.

## 4. 상호작용
*   **날짜/주/월 이동**: 캘린더 뷰에서 이전/다음 날짜, 주, 월로 이동합니다.
*   **일정 클릭**: 특정 일정을 클릭하면 상세 정보가 모달 또는 팝업으로 표시됩니다.
*   **새 일정 추가 버튼 클릭**: 일정 추가 폼이 있는 모달 또는 페이지로 이동합니다.
*   **일정 수정/삭제**: 그룹장/관리자는 각 일정 옆에 수정/삭제 버튼을 통해 관리할 수 있습니다.

## 5. 디자인 고려사항
*   **시각적 명확성**: 각 일정을 색상, 아이콘 등으로 구분하여 중요도나 종류를 쉽게 파악할 수 있도록 합니다.
*   **드래그 앤 드롭 (Post-MVP)**: 일정을 드래그 앤 드롭하여 시간을 변경하거나, 새로운 일정을 추가하는 기능을 고려할 수 있습니다.
*   **권한 관리**: 일정 추가, 수정, 삭제 버튼은 그룹장/관리자에게만 보이도록 처리합니다.
*   **빈 상태 처리**: 해당 날짜에 일정이 없을 경우, "등록된 일정이 없습니다."와 같은 안내 메시지를 표시합니다.
*   **시간대 관리**: 스터디 멤버들의 다양한 시간대를 고려하여 일정 표시 시 시간대 정보를 명확히 제공합니다.

## 6. UI 요소 상세 (Detailed UI Elements)

### 6.1. 상단 헤더 (Global Header) 및 좌측 사이드바 (Left Sidebar)
*   메인 대시보드 페이지와 동일한 구성 및 스타일을 따릅니다.

### 6.2. 스터디 헤더 (Study Header) 및 탭 네비게이션 (Tab Navigation)
*   스터디 개요 페이지와 동일한 구성 및 스타일을 따릅니다.

### 6.3. 캘린더 영역 (Calendar Area)
*   **구성**: 페이지 타이틀, 새 일정 추가 버튼, 캘린더 뷰, 일정 목록 (선택 사항).
*   **레이아웃**: 스터디 상세 콘텐츠 영역 내에서 상단에 타이틀과 버튼, 하단에 캘린더 뷰.
*   **스타일**: 배경 흰색 (#FFFFFF), 둥근 모서리 (8px), 가벼운 그림자 (shadow-sm), 패딩 20px.

### 6.4. 페이지 타이틀 (Page Title)
*   **텍스트**: "캘린더".
*   **스타일**: H3 태그, Noto Sans KR, 폰트 크기 24px, Neutral Dark Color, 좌측 정렬.

### 6.5. 새 일정 추가 버튼 (Add New Event Button)
*   **구성**: "새 일정 추가" 텍스트, 아이콘 (Material Icons - add).
*   **스타일**: Primary Button, 높이 40px, 너비 150px.
*   **위치**: 페이지 타이틀 우측.
*   **권한**: 그룹장/관리자에게만 표시.

### 6.6. 캘린더 뷰 (Calendar View)
*   **구성**: 월/주/일 전환 버튼, 이전/다음 버튼, 현재 날짜 표시, 날짜 그리드, 일정 표시.
*   **스타일**: FullCalendar 또는 유사한 라이브러리 기반. 각 날짜 셀은 스터디 일정을 색상으로 표시.
*   **일정 표시**: 일정 제목, 시간. 클릭 시 상세 모달.

### 6.7. 일정 상세 모달 (Event Detail Modal)
*   **구성**: 모달 헤더 (제목, 닫기 버튼), 시작/종료 시간, 설명, 작성자 정보, (그룹장/관리자에게) 수정/삭제 버튼.
*   **스타일**: 배경 흰색, 둥근 모서리, 그림자. 내용 영역은 스크롤 가능.

### 6.8. 빈 상태 (Empty State)
*   **구성**: 아이콘 (Material Icons - event_note), "등록된 일정이 없습니다." 텍스트.
*   **스타일**: 중앙 정렬, Neutral Gray Color.
