# 18. 스터디 상세 페이지 (Study Detail - Tasks Page)

## 1. 목적
스터디 그룹 멤버들이 스터디 관련 할 일(Tasks)을 생성, 관리하고 진행 상황을 공유할 수 있도록 합니다. 할 일은 스터디 캘린더와 연동되어 마감일을 기준으로 캘린더에서도 확인할 수 있습니다.

## 2. 주요 요소
*   **상단 헤더**: CoUp 로고, 사용자 프로필 이미지/닉네임, 알림 아이콘, 검색 바 (전역 헤더)
*   **좌측 사이드바**: 주요 메뉴 (전역 사이드바)
*   **스터디 헤더**: 스터디명, 스터디 소개 요약 등 (Overview 페이지와 동일)
*   **탭 네비게이션**: 스터디 내의 주요 기능(개요, 채팅, 공지, 파일, 캘린더, 할 일, 화상 스터디, 설정)으로 이동하는 탭 메뉴
*   **할 일 목록**: 스터디 관련 할 일들을 리스트 형태로 표시
*   **할 일 항목**:
    *   **체크박스**: 할 일 완료 여부 표시 및 변경
    *   **할 일 내용**: 할 일의 상세 내용
    *   **담당자**: 할 일을 담당하는 멤버의 프로필 이미지/닉네임 (선택 사항)
    *   **마감일**: 할 일의 마감일 (선택 사항, 캘린더에 표시됨)
    *   **작성자**: 할 일을 생성한 멤버의 닉네임
    *   **생성일**: 할 일이 생성된 날짜
    *   **수정/삭제 버튼**: (작성자 또는 그룹장/관리자에게만 표시) 할 일 수정 및 삭제 기능
*   **새 할 일 추가 입력 필드**: 새로운 할 일을 빠르게 추가할 수 있는 입력 필드 및 추가 버튼
*   **새 할 일 추가 입력 필드**: 새로운 할 일을 빠르게 추가할 수 있는 입력 필드 및 추가 버튼

## 3. 레이아웃
*   **2단 레이아웃**: 좌측 전역 사이드바, 우측 스터디 상세 콘텐츠 영역으로 구성됩니다.
*   **할 일 목록**: 스터디 상세 콘텐츠 영역 내에서 할 일 목록이 표시되며, 상단에 새 할 일 추가 입력 필드가 위치합니다.
*   **반응형 디자인**: 모바일 환경에서는 할 일 목록과 입력 필드가 화면에 최적화되어 표시됩니다.

## 4. 상호작용
*   **체크박스 클릭**: 할 일의 완료 상태를 토글합니다.
*   **새 할 일 추가**: 입력 필드에 내용을 입력하고 추가 버튼 클릭 시 새로운 할 일을 생성합니다. 마감일이 설정된 할 일은 캘린더에도 자동으로 반영됩니다.
*   **할 일 수정/삭제**: (권한이 있는 경우) 할 일 항목 옆의 수정/삭제 버튼 클릭 시 해당 할 일을 수정하거나 삭제합니다. 마감일 변경 시 캘린더에도 반영됩니다.
*   **담당자/마감일 설정**: 할 일 수정 시 담당자 및 마감일을 설정할 수 있는 모달 또는 팝업 제공.

## 5. 디자인 고려사항
*   **직관적인 완료 상태**: 체크박스를 통해 할 일의 완료 상태를 명확하게 표시하고, 완료된 할 일은 취소선 등으로 시각적 피드백을 제공합니다.
*   **권한 관리**: 수정/삭제 버튼은 해당 할 일의 작성자 또는 그룹장/관리자에게만 보이도록 처리합니다.
*   **빈 상태 처리**: 할 일이 없을 경우, "아직 등록된 할 일이 없습니다. 첫 할 일을 추가해보세요!"와 같은 안내 메시지를 표시합니다.
*   **필터링/정렬 (Post-MVP)**: 할 일 목록이 많아질 경우, 완료 여부, 담당자, 마감일 등으로 필터링하거나 정렬하는 기능을 고려할 수 있습니다.
*   **캘린더 연동 시각화**: 마감일이 있는 할 일은 캘린더에서 다른 일정과 구분되는 방식으로 시각화되어야 합니다.

## 6. UI 요소 상세 (Detailed UI Elements)

### 6.1. 상단 헤더 (Global Header) 및 좌측 사이드바 (Left Sidebar)
*   메인 대시보드 페이지와 동일한 구성 및 스타일을 따릅니다.

### 6.2. 스터디 헤더 (Study Header) 및 탭 네비게이션 (Tab Navigation)
*   스터디 개요 페이지와 동일한 구성 및 스타일을 따릅니다.

### 6.3. 할 일 관리 영역 (Task Management Area)
*   **구성**: 페이지 타이틀, 새 할 일 추가 입력 필드, 할 일 목록.
*   **레이아웃**: 스터디 상세 콘텐츠 영역 내에서 상단에 타이틀과 입력 필드, 하단에 할 일 목록.
*   **스타일**: 배경 흰색 (#FFFFFF), 둥근 모서리 (8px), 가벼운 그림자 (shadow-sm), 패딩 20px.

### 6.4. 페이지 타이틀 (Page Title)
*   **텍스트**: "할 일".
*   **스타일**: H3 태그, Noto Sans KR, 폰트 크기 24px, Neutral Dark Color, 좌측 정렬.

### 6.5. 새 할 일 추가 입력 필드 (Add New Task Input Field)
*   **구성**: Input 필드, "추가" 버튼.
*   **Input 필드**: 플레이스홀더 "새로운 할 일을 입력하세요.".
*   **추가 버튼**: Primary Button, 아이콘 (Material Icons - add).

### 6.6. 할 일 목록 (Task List)
*   **구성**: 개별 할 일 항목들의 리스트.
*   **레이아웃**: 각 할 일 항목이 세로로 나열되는 리스트 형태.
*   **스타일**: 각 항목은 하단 테두리 (Neutral Light Gray)로 구분.

### 6.7. 할 일 항목 (Task Item)
*   **구성**: 체크박스, 할 일 내용, 담당자 (프로필 이미지/닉네임), 마감일, 작성자, 생성일, 수정/삭제 버튼.
*   **체크박스**: 할 일 내용 좌측에 배치.
*   **할 일 내용**: 폰트 16px, Neutral Dark Color. 완료 시 취소선 적용.
*   **담당자/마감일/작성자/생성일**: 폰트 12px, Neutral Gray Color.
*   **수정/삭제 버튼**: Material Icons (edit, delete), 작은 크기, Secondary Outline Button 스타일. Hover 시 색상 변화.
*   **권한**: 수정/삭제 버튼은 작성자 또는 그룹장/관리자에게만 표시.
