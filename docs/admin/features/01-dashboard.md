# 관리자 대시보드 - 기능 명세

> **영역**: 대시보드  
> **우선순위**: P0 (최우선)  
> **담당**: Admin System

---

## 📊 개요

관리자 대시보드는 플랫폼의 전체 현황을 한눈에 파악할 수 있는 메인 화면입니다.

---

## 🎯 핵심 목표

1. **실시간 모니터링**: 플랫폼 주요 지표 실시간 표시
2. **빠른 의사결정**: 이상 징후 즉시 감지 및 알림
3. **효율적인 워크플로우**: 주요 작업 바로가기 제공

---

## 📋 기능 목록

### 1. 핵심 지표 카드 (Stat Cards)

#### 1.1 사용자 지표 카드
```
┌─────────────────────────────┐
│  👥 사용자                   │
│                             │
│  1,234  (+12 오늘)          │
│  ───────────────────        │
│  활성: 856  정지: 3          │
└─────────────────────────────┘
```

**표시 정보**:
- 총 사용자 수 (큰 숫자)
- 오늘 신규 가입자 수 (작은 텍스트, 초록색)
- 활성 사용자 수 (7일 기준)
- 정지 사용자 수 (빨간색)

**인터랙션**:
- 클릭 시 → 사용자 관리 페이지로 이동

#### 1.2 스터디 지표 카드
```
┌─────────────────────────────┐
│  📚 스터디                   │
│                             │
│  542  (+5 오늘)             │
│  ───────────────────        │
│  활성: 312  모집중: 189      │
└─────────────────────────────┘
```

**표시 정보**:
- 총 스터디 수
- 오늘 신규 생성 수
- 활성 스터디 수 (7일 내 활동)
- 모집 중인 스터디 수

**인터랙션**:
- 클릭 시 → 스터디 관리 페이지로 이동

#### 1.3 신고 지표 카드
```
┌─────────────────────────────┐
│  ⚠️ 신고                    │
│                             │
│  23  (미처리)               │
│  ───────────────────        │
│  긴급: 2  오늘: 8            │
└─────────────────────────────┘
```

**표시 정보**:
- 미처리 신고 수 (대기 + 처리중)
- 긴급 신고 수 (빨간색 강조)
- 오늘 접수된 신고 수

**인터랙션**:
- 클릭 시 → 신고 관리 페이지 (미처리 필터)

**알림**:
- 긴급 신고가 1개 이상이면 빨간 점 표시
- 미처리 신고가 50개 이상이면 경고 표시

#### 1.4 콘텐츠 지표 카드
```
┌─────────────────────────────┐
│  💬 활동                     │
│                             │
│  12,345  (메시지)           │
│  ───────────────────        │
│  파일: 1,234  공지: 456      │
└─────────────────────────────┘
```

**표시 정보**:
- 총 메시지 수
- 총 파일 수
- 총 공지 수

---

### 2. 실시간 활동 피드

#### 2.1 최근 가입 사용자
```
┌─────────────────────────────────────────┐
│  최근 가입 사용자                        │
├─────────────────────────────────────────┤
│  [👤] 홍길동 (hong@example.com)         │
│       3분 전 · Google 로그인             │
│                                         │
│  [👤] 김철수 (kim@example.com)          │
│       15분 전 · 이메일 회원가입          │
│                                         │
│  [더보기 →]                             │
└─────────────────────────────────────────┘
```

**표시 정보**:
- 프로필 사진 (또는 기본 아이콘)
- 이름, 이메일
- 가입 시간 (상대 시간)
- 가입 방법

**개수**: 최근 5명

**인터랙션**:
- 이름 클릭 → 사용자 상세 페이지
- [더보기] 클릭 → 사용자 목록 (최신순 정렬)

#### 2.2 최근 생성 스터디
```
┌─────────────────────────────────────────┐
│  최근 생성 스터디                        │
├─────────────────────────────────────────┤
│  📚 알고리즘 스터디                      │
│      by 홍길동 · 1시간 전                │
│                                         │
│  🎨 디자인 스터디                        │
│      by 김철수 · 3시간 전                │
│                                         │
│  [더보기 →]                             │
└─────────────────────────────────────────┘
```

**표시 정보**:
- 이모지, 스터디 이름
- 생성자, 생성 시간

**개수**: 최근 5개

**인터랙션**:
- 스터디 이름 클릭 → 스터디 상세 페이지
- [더보기] 클릭 → 스터디 목록 (최신순 정렬)

#### 2.3 최근 접수 신고
```
┌─────────────────────────────────────────┐
│  최근 접수 신고                          │
├─────────────────────────────────────────┤
│  [🔴 긴급] 사용자 신고: 욕설              │
│      5분 전 · 신고자: 익명               │
│                                         │
│  [⚠️ 높음] 스터디 신고: 스팸              │
│      20분 전 · 신고자: 홍길동            │
│                                         │
│  [더보기 →]                             │
└─────────────────────────────────────────┘
```

**표시 정보**:
- 우선순위 아이콘 + 텍스트
- 신고 유형 + 사유 (한 줄)
- 접수 시간, 신고자

**개수**: 최근 10개

**색상**:
- 긴급: 빨간색 배경
- 높음: 노란색 배경
- 보통: 기본

**인터랙션**:
- 신고 클릭 → 신고 상세 페이지
- [더보기] 클릭 → 신고 목록

### 3. 차트 및 그래프

#### 3.1 사용자 추이 차트
```
사용자 활동 추이 (최근 30일)
┌─────────────────────────────────┐
│ 100 ┤                        ╭─╮│
│  90 ┤                    ╭───╯ ││
│  80 ┤                ╭───╯     ││
│  70 ┤            ╭───╯         ││
│  60 ┤        ╭───╯             ││
│  50 ┤    ╭───╯                 ││
│  40 ┤╭───╯                     ││
│     └┬───┬───┬───┬───┬───┬───┬┘│
│      11/1  11/8  11/15 11/22   │
│                                 │
│  ─── 신규 가입  ─── 활성 사용자  │
└─────────────────────────────────┘
```

**차트 유형**: Line Chart (라인 차트)

**데이터**:
- 신규 가입자 수 (일별)
- 활성 사용자 수 (일별)

**기간**: 최근 30일

**인터랙션**:
- 호버 시 → 해당 날짜의 정확한 수치 표시
- 범례 클릭 → 데이터 시리즈 on/off

#### 3.2 신고 처리 현황 차트
```
신고 처리 현황
┌─────────────────────────────────┐
│          ╱╲                     │
│         ╱  ╲                    │
│        ╱ 23 ╲                   │
│       ╱ 대기 ╲                  │
│      ╱────────╲                 │
│     │  15 처리중 │               │
│     │──────────│               │
│      ╲   42   ╱                 │
│       ╲ 완료 ╱                  │
│        ╲    ╱                   │
│         ╲  ╱                    │
│          ╲╱                     │
│                                 │
│  ■ 대기  ■ 처리중  ■ 완료  ■ 기각│
└─────────────────────────────────┘
```

**차트 유형**: Donut Chart (도넛 차트)

**데이터**:
- 대기: 노란색
- 처리중: 파란색
- 완료: 초록색
- 기각: 회색

**인터랙션**:
- 섹션 클릭 → 해당 상태의 신고 목록

#### 3.3 인기 카테고리 차트
```
인기 스터디 카테고리 (Top 5)
┌─────────────────────────────────┐
│ 개발     ████████████████  142   │

│ 자격증   ██████  58               │

└─────────────────────────────────┘
```

**차트 유형**: Bar Chart (막대 차트)

**데이터**:
- 카테고리별 스터디 수 (Top 5)

**인터랙션**:
- 막대 클릭 → 해당 카테고리의 스터디 목록

### 4. 빠른 액션 버튼

#### 4.1 주요 작업 바로가기
```
┌─────────────────────────────────────────┐
│  빠른 작업                               │
├─────────────────────────────────────────┤
│  [➕ 시스템 공지 발송]                   │
│  [👥 사용자 검색]                        │
│  [📚 스터디 검색]                        │
│  [📊 리포트 생성]                        │
└─────────────────────────────────────────┘
```

**버튼 목록**:
1. **시스템 공지 발송**: 모달 오픈
2. **사용자 검색**: 사용자 목록 페이지 (검색창 포커스)
3. **스터디 검색**: 스터디 목록 페이지 (검색창 포커스)
4. **리포트 생성**: 리포트 생성 모달

---

### 5. 시스템 상태

#### 5.1 서버 상태
```
┌─────────────────────────────────────────┐
│  시스템 상태                             │
├─────────────────────────────────────────┤
│  [✅] 웹 서버: 정상                      │
│  [✅] 데이터베이스: 정상                 │
│  [✅] 시그널링 서버: 정상                │
│  [⚠️] 스토리지: 85% 사용 중              │
│                                         │
│  최종 점검: 5분 전                       │
└─────────────────────────────────────────┘
```

**표시 정보**:
- 각 서비스 상태 (정상/경고/에러)
- 스토리지 사용률
- 최종 점검 시간

**상태 색상**:
- ✅ 정상: 초록색
- ⚠️ 경고: 노란색
- ❌ 에러: 빨간색

## 📱 반응형 디자인

### Breakpoints
- **Desktop**: 1200px 이상
- **Tablet**: 768px ~ 1199px
- **Mobile**: 767px 이하

### 지표 카드 그리드
- Desktop: 4열 (1행에 4개)
- Tablet: 2열 (1행에 2개)
- Mobile: 1열 (세로 스택)

### 차트 레이아웃
- Desktop: 2열 (가로 나란히)
- Tablet/Mobile: 1열 (세로 스택)

---

## 🔄 데이터 갱신

### 자동 갱신
- **지표 카드**: 30초마다 자동 갱신
- **실시간 활동 피드**: 10초마다 자동 갱신
- **차트**: 5분마다 자동 갱신

### 수동 갱신
- 우측 상단 [새로고침] 버튼
- 각 위젯별 개별 새로고침 버튼

## 🛠️ 기술 스택

### 프론트엔드
- **React**: UI 컴포넌트
- **Chart.js / Recharts**: 차트 라이브러리
- **SWR / React Query**: 데이터 페칭 및 캐싱

### 백엔드 API
- **GET /api/admin/dashboard/stats**: 핵심 지표
- **GET /api/admin/dashboard/recent-users**: 최근 가입 사용자
- **GET /api/admin/dashboard/recent-studies**: 최근 생성 스터디
- **GET /api/admin/dashboard/recent-reports**: 최근 신고
- **GET /api/admin/dashboard/chart-data**: 차트 데이터
