## 1. 대시보드

### 1.1 핵심 지표 카드

#### 사용자 지표
- **총 사용자 수**: 전체 가입자 수
- **활성 사용자 수**: 최근 7일 내 활동한 사용자
- **신규 가입자**: 오늘/이번 주/이번 달
- **정지 사용자**: 현재 정지 상태인 사용자

#### 스터디 지표
- **총 스터디 수**: 전체 생성된 스터디
- **활성 스터디 수**: 최근 7일 내 활동이 있는 스터디
- **신규 스터디**: 오늘/이번 주/이번 달
- **평균 멤버 수**: 스터디당 평균 인원

#### 콘텐츠 지표
- **총 메시지 수**: 전체 채팅 메시지
- **총 파일 수**: 업로드된 파일 수
- **총 공지 수**: 작성된 공지사항 수

#### 신고 지표
- **미처리 신고**: 대기/처리중 상태
- **오늘 접수 신고**: 오늘 들어온 신고
- **긴급 신고**: 높은 우선순위 신고

### 1.2 실시간 활동 피드

#### 표시 내용
- 최근 가입 사용자 (최근 10명)
- 최근 생성 스터디 (최근 10개)
- 최근 접수 신고 (최근 20개)
- 시스템 이벤트 (에러, 경고)

### 1.3 차트 및 그래프

#### 사용자 추이 (라인 차트)
- X축: 날짜 (최근 30일)
- Y축: 사용자 수
- 데이터: 신규 가입, 활성 사용자, 정지 사용자

#### 스터디 추이 (라인 차트)
- X축: 날짜 (최근 30일)
- Y축: 스터디 수
- 데이터: 신규 생성, 활성 스터디, 종료 스터디

#### 신고 처리 현황 (도넛 차트)
- 대기: 노란색
- 처리중: 파란색
- 완료: 초록색
- 기각: 회색

#### 인기 카테고리 (막대 차트)
- X축: 카테고리명
- Y축: 스터디 수

## 2. 사용자 관리

### 2.1 사용자 목록

#### 필터 옵션
- **상태**: 전체, 활성, 정지, 탈퇴
- **역할**: 전체, USER, ADMIN, SYSTEM_ADMIN
- **가입 방법**: 전체, 이메일, Google, GitHub
- **가입일**: 전체, 오늘, 이번 주, 이번 달, 기간 지정

#### 정렬 옵션
- 가입일 (최신순/오래된순)
- 최근 로그인 (최신순/오래된순)
- 이름 (가나다순/역순)
- 스터디 참여 수 (많은순/적은순)

#### 검색
- 이메일, 이름으로 검색
- 자동완성 지원

#### 표시 정보 (테이블)
| 항목 | 설명 |
|------|------|
| 프로필 사진 | 썸네일 |
| 이름 | 클릭 시 상세 페이지 |
| 이메일 | 부분 마스킹 (예: t***@example.com) |
| 상태 | 배지 (활성/정지/탈퇴) |
| 역할 | 배지 (USER/ADMIN/SYSTEM_ADMIN) |
| 가입일 | YYYY-MM-DD |
| 최근 로그인 | 상대 시간 (예: 3시간 전) |
| 스터디 수 | 참여 중인 스터디 개수 |
| 액션 | 보기, 정지, 편집 버튼 |

#### 일괄 작업
- 선택한 사용자 일괄 정지
- 선택한 사용자 일괄 알림 발송

### 2.2 사용자 상세

#### 기본 정보
- 프로필 사진
- 이름, 이메일
- 자기소개
- 가입 방법 (이메일/Google/GitHub)
- 가입일, 최근 로그인

#### 활동 통계
- 참여 스터디 수
- 보낸 메시지 수
- 업로드한 파일 수
- 작성한 공지 수

#### 스터디 목록
- 참여 중인 스터디 (역할 표시)
- 만든 스터디
- 탈퇴한 스터디 (이력)

#### 제재 이력
| 날짜 | 유형 | 사유 | 기간 | 관리자 |
|------|------|------|------|--------|
| 2025-11-20 | 정지 | 욕설 | 3일 | admin@coup.com |
| 2025-11-15 | 경고 | 스팸 | - | admin@coup.com |

#### 신고 이력
- 신고한 신고 (신고자로서)
- 신고당한 신고 (피신고자로서)

#### 관리 액션
- **경고 발송**: 모달로 사유 입력
- **정지 처리**: 정지 기간 및 사유 입력
- **정지 해제**: 확인 후 즉시 해제
- **역할 변경**: SYSTEM_ADMIN만 가능
- **탈퇴 처리**: SYSTEM_ADMIN만 가능

### 2.3 사용자 정지

#### 정지 모달
```
┌─────────────────────────────────┐
│  사용자 정지                     │
├─────────────────────────────────┤
│                                 │
│  사용자: 홍길동 (hong@example.com)│
│                                 │
│  정지 기간:                      │
│  [ ] 1일   [ ] 3일   [ ] 7일     │
│  [ ] 30일  [ ] 영구              │
│                                 │
│  정지 사유: (필수)               │
│  ┌─────────────────────────┐   │
│  │                         │   │
│  │  최소 10자 이상 입력     │   │
│  │                         │   │
│  └─────────────────────────┘   │
│                                 │
│  관련 신고 선택: (선택)          │
│  [ ] 신고 #123 - 욕설            │
│  [ ] 신고 #124 - 스팸            │
│                                 │
│  [ ] 사용자에게 이메일 발송      │
│                                 │
│  [취소]              [정지하기]  │
└─────────────────────────────────┘
```

## 3. 스터디 관리

### 3.1 스터디 목록

#### 필터 옵션
- **상태**: 전체, 모집중, 모집완료, 종료, 숨김
- **카테고리**: 전체, 개발, 디자인, 어학 등
- **공개 여부**: 전체, 공개, 비공개
- **생성일**: 전체, 오늘, 이번 주, 이번 달, 기간 지정

#### 정렬 옵션
- 생성일 (최신순/오래된순)
- 멤버 수 (많은순/적은순)
- 평점 (높은순/낮은순)
- 최근 활동 (최신순/오래된순)

#### 검색
- 스터디 이름, 설명, 태그로 검색

#### 표시 정보 (카드 또는 테이블)
| 항목 | 설명 |
|------|------|
| 이모지 | 스터디 대표 이모지 |
| 스터디 이름 | 클릭 시 상세 페이지 |
| 설명 | 한 줄 요약 (최대 100자) |
| 카테고리 | 배지 |
| 멤버 수 | 현재/최대 (예: 5/20) |
| 상태 | 배지 (모집중/종료/숨김) |
| 생성자 | 이름 + 프로필 사진 |
| 생성일 | YYYY-MM-DD |
| 최근 활동 | 상대 시간 |
| 액션 | 보기, 숨김, 종료 버튼 |

### 3.2 스터디 상세

#### 기본 정보
- 이모지, 이름
- 설명
- 카테고리, 태그
- 생성자 정보
- 생성일, 최근 활동

#### 통계
- 총 멤버 수
- 역할별 멤버 수 (OWNER/ADMIN/MEMBER)
- 총 메시지 수
- 총 파일 수
- 평균 활동도

#### 멤버 목록
| 이름 | 역할 | 가입일 | 메시지 수 | 상태 |
|------|------|--------|----------|------|
| 홍길동 | OWNER | 2025-11-01 | 152 | 활성 |
| 김철수 | ADMIN | 2025-11-05 | 98 | 활성 |

#### 최근 메시지 (최근 50개)
- 작성자, 내용, 시간
- 부적절한 메시지 필터링 (빨간색 강조)

#### 업로드 파일
- 파일명, 크기, 업로더, 업로드일
- 다운로드 수

#### 관리 액션
- **숨김 처리**: 검색 결과에서 제외
- **모집 중단**: 신규 멤버 가입 차단
- **스터디 종료**: 강제 종료
- **추천 설정**: 추천 스터디로 선정
- **메시지 삭제**: 부적절한 메시지 선택 삭제
- **파일 삭제**: 불법 파일 삭제

### 3.3 스터디 종료

#### 종료 모달
```
┌─────────────────────────────────┐
│  스터디 종료                     │
├─────────────────────────────────┤
│                                 │
│  스터디: 📚 알고리즘 스터디       │
│  멤버 수: 15명                   │
│                                 │
│  종료 사유: (필수)               │
│  ┌─────────────────────────┐   │
│  │                         │   │
│  │  부적절한 콘텐츠         │   │
│  │                         │   │
│  └─────────────────────────┘   │
│                                 │
│  [ ] 멤버에게 알림 발송          │
│  [ ] 검색 결과에서 숨김          │
│                                 │
│  ⚠️ 이 작업은 되돌릴 수 없습니다.│
│                                 │
│  [취소]              [종료하기]  │
└─────────────────────────────────┘
```

## 4. 신고 관리

### 4.1 신고 목록

#### 필터 옵션
- **상태**: 전체, 대기, 처리중, 완료, 기각
- **유형**: 전체, 사용자, 스터디, 메시지, 공지, 파일
- **우선순위**: 전체, 긴급, 높음, 보통, 낮음
- **접수일**: 전체, 오늘, 이번 주, 이번 달

#### 정렬 옵션
- 접수일 (최신순/오래된순)
- 우선순위 (높은순/낮은순)

#### 표시 정보 (테이블)
| 항목 | 설명 |
|------|------|
| ID | 신고 고유 번호 |
| 유형 | 배지 (사용자/스터디/콘텐츠) |
| 신고 내용 | 한 줄 요약 |
| 신고자 | 이름 (익명 선택 시 '익명') |
| 피신고자/대상 | 이름 또는 스터디명 |
| 우선순위 | 배지 (긴급/높음/보통/낮음) |
| 상태 | 배지 (대기/처리중/완료/기각) |
| 접수일 | 상대 시간 |
| 담당자 | 관리자 이름 (미배정 시 '-') |
| 액션 | 보기, 처리 버튼 |

### 4.2 신고 상세

#### 신고 정보
- 신고 ID, 접수일
- 신고 유형, 우선순위
- 신고자 정보 (이름, 이메일)
- 피신고자/대상 정보

#### 신고 내용
- 신고 사유 (선택한 유형)
- 상세 설명
- 첨부 증거 (스크린샷, 링크 등)

#### 대상 미리보기
- **사용자**: 프로필, 최근 활동
- **스터디**: 기본 정보, 멤버 수
- **메시지/공지**: 원본 내용
- **파일**: 파일 정보

#### 관련 이력
- 피신고자의 이전 신고 이력
- 유사한 신고 건

#### 관리자 작업 영역
```
┌─────────────────────────────────┐
│  처리 상태 변경                  │
│  [ ] 대기  [ ] 처리중  [ ] 완료  │
│                                 │
│  우선순위 변경                   │
│  [ ] 긴급  [ ] 높음  [ ] 보통    │
│                                 │
│  담당자 배정                     │
│  [드롭다운: 관리자 목록]         │
│                                 │
│  관리자 메모                     │
│  ┌─────────────────────────┐   │
│  │                         │   │
│  │  내부용 메모 작성        │   │
│  │                         │   │
│  └─────────────────────────┘   │
│                                 │
│  [메모 저장]                     │
└─────────────────────────────────┘
```

#### 처리 액션
- **승인**: 제재 조치 실행
- **기각**: 신고 내용 부적절
- **보류**: 추가 조사 필요
- **에스컬레이션**: SYSTEM_ADMIN에게 전달

### 4.3 신고 처리

#### 승인 모달
```
┌─────────────────────────────────┐
│  신고 승인 및 제재 실행           │
├─────────────────────────────────┤

│  제재 조치:                      │
│  [ ] 경고                        │

│  [ ] 스터디 종료                 │
│                                 │
│  제재 사유: (자동 입력)          │

│  │ 욕설 및 혐오 발언        │   │

│  └─────────────────────────┘   │
│                                 │
│  [ ] 신고자에게 알림 발송         │
│  [ ] 피신고자에게 알림 발송       │
│                                 │
│  [취소]              [실행하기]  │
└─────────────────────────────────┘
```

## 5. 콘텐츠 검열

### 5.1 금지어 필터링

#### 자동 필터링
- 설정된 금지어 포함 콘텐츠 자동 감지
- 관리자 검토 대기열에 추가

#### 수동 검토
- 감지된 콘텐츠 목록
- 오탐 여부 확인
- 승인/삭제 결정

### 5.2 이미지 검열

#### 부적절한 프로필 사진
- 자동 AI 필터 (NSFW 탐지)
- 관리자 수동 검토
- 삭제 및 기본 이미지로 변경

#### 업로드 파일
- 불법 파일 확장자 차단
- 파일 내용 스캔 (바이러스, 불법 콘텐츠)

## 6. 통계 및 분석

### 6.1 사용자 분석

#### 가입 추이
- 일별/주별/월별 신규 가입자 수
- 소셜 로그인 비율

#### 활동 분석
- DAU (Daily Active Users)
- MAU (Monthly Active Users)
- 활동률 (활성 사용자 / 전체 사용자)

#### 유지율 (Retention)
- D1, D7, D30 유지율
- 코호트 분석

### 6.2 스터디 분석

#### 생성 추이
- 일별/주별/월별 신규 스터디 수

#### 카테고리별 분포
- 인기 카테고리
- 카테고리별 평균 멤버 수

#### 품질 분석
- 평균 멤버 수
- 평균 메시지 수
- 평균 활동 기간

### 6.3 리포트 생성

#### 리포트 유형
1. **일간 리포트**: 오늘의 주요 지표
2. **주간 리포트**: 이번 주 요약
3. **월간 리포트**: 이번 달 종합 분석

#### 리포트 내용
- 사용자 지표

- 신고 처리 현황
- 주요 이슈 및 조치

- PDF 다운로드

## 7. 시스템 설정

### 7.1 플랫폼 설정

#### 회원가입 제한
- 이메일 도메인 화이트리스트
- 최소 연령 제한
- 초대 코드 필수 여부

#### 스터디 생성 제한
- 최소 가입 기간 (예: 7일 이상)
- 최대 생성 개수 (예: 사용자당 5개)

#### 파일 업로드 제한
- 최대 파일 크기 (예: 10MB)
- 허용 확장자
- 바이러스 스캔 활성화

### 7.2 자동 필터링 규칙

#### 금지어 관리
- 금지어 목록 추가/수정/삭제
- 필터링 강도 (낮음/보통/높음)

#### 스팸 탐지
- 반복 메시지 감지 (동일 내용 N회 이상)
- 링크 스팸 탐지
- 계정 생성 후 즉시 광고 차단

### 7.3 이메일 설정

#### SMTP 설정
- 발송 서버, 포트
- 인증 정보

#### 이메일 템플릿
- 회원가입 환영
- 비밀번호 재설정
- 제재 알림
- 시스템 공지

## 8. 알림 관리

### 8.1 시스템 공지

#### 공지 생성
- 제목, 내용
- 발송 대상 (전체/특정 그룹)
- 발송 시간 (즉시/예약)

#### 긴급 알림
- 긴급 배지
- 우선 표시
- 푸시 알림 강제 발송

### 8.2 자동 알림

#### 트리거 설정
- 신고 접수 시 관리자에게
- 신규 가입 시 관리자에게
- 시스템 에러 시 관리자에게
