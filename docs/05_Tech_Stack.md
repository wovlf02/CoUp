# 05. 기술 스택 (Technology Stack)

CoUp 프로젝트는 각 영역에서 가장 안정적이고 성능이 검증된 기술을 사용하여, 확장성 높고 유지보수하기 좋은 시스템을 구축하는 것을 목표로 합니다. 프론트엔드, 백엔드, 시그널링 서버는 명확히 분리된 프로젝트로 개발됩니다.

## 1. 프론트엔드 (Frontend)

| 구분 | 기술 | 버전/라이브러리 | 선택 이유 |
| :--- | :--- | :--- | :--- |
| **언어** | **JavaScript (ES2022+)** | - | 웹 표준 언어로서 방대한 생태계와 자료를 보유하고 있으며, 빠른 UI 개발에 유리합니다. | 
| **프레임워크** | **Next.js** | 15.x | React 기반의 프레임워크. SSR, 라우팅 등 현대적인 프론트엔드 개발에 필요한 기능을 제공합니다. | 
| **UI 라이브러리** | **React** | 18.x | 컴포넌트 기반 아키텍처의 표준. 풍부한 생태계를 바탕으로 복잡한 UI를 효율적으로 구축합니다. | 
| **상태 관리** | **Zustand** | 4.x | 간결하고 직관적인 API를 제공하는 경량 상태 관리 라이브러리. | 
| **데이터 페칭** | **React Query (TanStack Query)** | 5.x | 서버 상태(Server State) 관리를 위한 필수 라이브러리. 캐싱, 동기화 등 복잡한 비동기 로직을 선언적으로 관리합니다. | 
| **스타일링** | **Tailwind CSS** | 3.x | Utility-First CSS 프레임워크. 디자인 시스템을 빠르게 구축하고, 일관성 있는 UI를 유지합니다. | 

## 2. 비즈니스 서버 (Backend)

| 구분 | 기술 | 버전/라이브러리 | 선택 이유 |
| :--- | :--- | :--- | :--- |
| **언어** | **Java** | 21.0.8 | 오랜 기간 안정성이 검증되었으며, 강력한 객체 지향 프로그래밍과 대규모 애플리케이션 구축에 적합합니다. | 
| **프레임워크** | **Spring Boot** | 3.5.7 | Java 기반 웹 애플리케이션 개발의 사실상 표준. DI, AOP 등을 통해 생산성과 확장성이 매우 뛰어납니다. | 
| **빌드 도구** | **Gradle** | 8.x | Groovy 또는 Kotlin DSL을 사용하여 유연하고 간결하게 빌드 스크립트를 작성할 수 있습니다. | 
| **데이터베이스 연동** | **MyBatis** | 3.x | SQL 매핑 프레임워크. XML 또는 어노테이션을 통해 SQL 쿼리를 소스 코드와 분리하고, 개발자가 SQL을 직접 제어하여 복잡한 쿼리나 최적화를 수행하기 용이합니다. | 
| **인증/인가** | **Spring Security** | - | 강력하고 유연한 인증/인가 프레임워크. JWT 기반 인증, 소셜 로그인 등 다양한 보안 요구사항을 처리할 수 있습니다. | 
| **API 문서화** | **SpringDoc (Swagger)** | - | 코드에 어노테이션을 추가하여 API 문서를 자동으로 생성하고, UI를 통해 API를 직접 테스트해볼 수 있습니다. | 

## 3. 시그널링 서버 (Signaling Server)

| 구분 | 기술 | 버전/라이브러리 | 선택 이유 |
| :--- | :--- | :--- | :--- |
| **언어** | **Node.js** | 18.x+ | 비동기 I/O 처리에 특화되어 있어, 다수의 WebSocket 연결을 효율적으로 관리하는 데 최적화되어 있습니다. | 
| **프레임워크** | **Express.js** | 4.x | Node.js 웹 서버 개발을 위한 최소한의 경량 프레임워크. | 
| **WebSocket 라이브러리** | **Socket.IO** | 4.x | 자동 재연결, 브로드캐스팅, 룸(Room) 기능 등 실시간 채팅 및 알림에 필요한 필수 기능을 안정적으로 제공합니다. | 
| **인증** | **jsonwebtoken** | 9.x | 클라이언트로부터 받은 JWT를 시그널링 서버에서 직접 검증하여, 인증된 사용자만 WebSocket 통신을 허용하도록 합니다. | 

## 4. 데이터베이스 및 인프라 (Database & Infra)

| 구분 | 기술 | 선택 이유 |
| :--- | :--- | :--- |
| **주요 RDBMS** | **PostgreSQL** | 15.x | 뛰어난 안정성, 성능, 풍부한 기능을 갖춘 오픈소스 관계형 데이터베이스. | 
| **캐시 & 메시지 브로커** | **Redis** | 7.x | **캐시**: DB 부하 감소 및 응답 속도 향상. **메시지 브로커(Pub/Sub)**: 비즈니스 서버와 시그널링 서버 간의 이벤트 기반 통신을 위해 사용. | 
| **파일 스토리지** | **AWS S3** | 안정성, 확장성, 보안성이 뛰어난 객체 스토리지. | 
| **컨테이너화** | **Docker** | 개발, 테스트, 프로덕션 환경을 일치시키고, 서비스의 격리 및 이식성을 보장합니다. | 
| **프론트엔드 호스팅** | **Vercel** | Next.js에 최적화된 배포, CI/CD, CDN을 제공합니다. | 
| **백엔드/시그널링 호스팅** | **AWS (ECS/Fargate)** | Docker 컨테이너 기반의 서버들을 안정적으로 운영하고, 트래픽에 따라 자동으로 확장하기 용이합니다. | 
| **CI/CD** | **GitHub Actions** | 프론트엔드, 백엔드, 시그널링 서버의 테스트, 빌드, 배포 파이프라인을 자동화합니다. |
