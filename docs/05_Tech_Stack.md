# 05. 기술 스택 (Technology Stack)

CoUp 프로젝트는 각 영역에서 가장 안정적이고 성능이 검증된 기술을 사용하여, 확장성 높고 유지보수하기 좋은 시스템을 구축하는 것을 목표로 합니다. Next.js가 프론트엔드와 백엔드를 모두 담당하며, 시그널링 서버는 별도로 분리된 프로젝트로 개발됩니다.

## 1. Next.js 풀스택 (Next.js Fullstack)

| 구분 | 기술 | 버전/라이브러리 | 선택 이유 |
| :--- | :--- | :--- | :--- |
| **언어** | **TypeScript** | - | JavaScript의 상위 집합으로, 정적 타입 검사를 통해 대규모 애플리케이션의 안정성과 유지보수성을 높입니다. | 
| **프레임워크** | **Next.js** | 15.x | React 기반의 풀스택 프레임워크. SSR, SSG, ISR 등 다양한 렌더링 방식과 API Routes를 통한 백엔드 기능 제공으로 생산성과 확장성이 뛰어납니다. | 
| **UI 라이브러리** | **React** | 18.x | 컴포넌트 기반 아키텍처의 표준. 풍부한 생태계를 바탕으로 복잡한 UI를 효율적으로 구축합니다. | 
| **상태 관리** | **Zustand** | 4.x | 간결하고 직관적인 API를 제공하는 경량 상태 관리 라이브러리. | 
| **데이터 페칭** | **React Query (TanStack Query)** | 5.x | 서버 상태(Server State) 관리를 위한 필수 라이브러리. 캐싱, 동기화 등 복잡한 비동기 로직을 선언적으로 관리합니다. | 
| **인증** | **NextAuth.js** | 4.x | Next.js 애플리케이션을 위한 강력하고 유연한 인증 라이브러리. 소셜 로그인, JWT 기반 세션 관리 등을 쉽게 구현할 수 있습니다. | 
| **ORM** | **Prisma** | 5.x | 현대적인 Node.js 및 TypeScript ORM. 타입 안전성을 제공하며, 데이터베이스 스키마 관리 및 쿼리 작성을 간소화합니다. | 

## 2. 시그널링 서버 (Signaling Server)

| 구분 | 기술 | 버전/라이브러리 | 선택 이유 |
| :--- | :--- | :--- | :--- |
| **언어** | **Node.js** | 18.x+ | 비동기 I/O 처리에 특화되어 있어, 다수의 WebSocket 연결을 효율적으로 관리하는 데 최적화되어 있습니다. Next.js API Routes와 연동하여 실시간 기능을 제공합니다. | 
| **프레임워크** | **Express.js** | 4.x | Node.js 웹 서버 개발을 위한 최소한의 경량 프레임워크. Next.js API Routes와의 내부 통신을 담당합니다. | 
| **WebSocket 라이브러리** | **Socket.IO** | 4.x | 자동 재연결, 브로드캐스팅, 룸(Room) 기능 등 실시간 채팅 및 알림에 필요한 필수 기능을 안정적으로 제공합니다. | 
| **인증** | **jsonwebtoken** | 9.x | 클라이언트로부터 받은 JWT를 시그널링 서버에서 직접 검증하여, 인증된 사용자만 WebSocket 통신을 허용하도록 합니다. NextAuth.js에서 발급한 JWT와 연동됩니다. | 

## 3. 데이터베이스 및 인프라 (Database & Infra)

| 구분 | 기술 | 선택 이유 |
| :--- | :--- | :--- |
| **주요 RDBMS** | **PostgreSQL** | 15.x | 뛰어난 안정성, 성능, 풍부한 기능을 갖춘 오픈소스 관계형 데이터베이스. | 
| **캐시 & 메시지 브로커** | **Redis** | 7.x | **캐시**: Next.js API Routes의 DB 부하 감소 및 응답 속도 향상. **메시지 브로커(Pub/Sub)**: Next.js API Routes와 시그널링 서버 간의 이벤트 기반 통신을 위해 사용. | 
| **파일 스토리지** | **AWS S3** | 안정성, 확장성, 보안성이 뛰어난 객체 스토리지. | 
| **컨테이너화** | **Docker** | 개발, 테스트, 프로덕션 환경을 일치시키고, 서비스의 격리 및 이식성을 보장합니다. | 
| **호스팅** | **Vercel (Next.js 풀스택)** | Next.js에 최적화된 배포, CI/CD, CDN을 제공하며, Next.js API Routes를 포함한 풀스택 애플리케이션을 호스팅합니다. | 
| **시그널링 서버 호스팅** | **AWS (ECS/Fargate)** | Docker 컨테이너 기반의 시그널링 서버를 안정적으로 운영하고, 트래픽에 따라 자동으로 확장하기 용이합니다. | 
| **CI/CD** | **GitHub Actions** | Next.js 풀스택 및 시그널링 서버의 테스트, 빌드, 배포 파이프라인을 자동화합니다. |
