# 14. 프로젝트 로드맵 및 마일스톤

본 로드맵은 CoUp 프로젝트의 개발 및 출시까지의 과정을 단계별로 정의합니다. 프론트엔드(Next.js), 비즈니스 서버(Spring Boot), 시그널링 서버(Node.js)는 분리된 프로젝트로 병렬 개발되는 것을 전제로 합니다.

## 전체 타임라인 요약

- **Phase 0: 준비 및 설계 (완료)**
- **Phase 1: 프로젝트 기반 구축 및 인증 개발 (2주)**
- **Phase 2: MVP 핵심 기능 개발 (5주)**
- **Phase 3: MVP 테스트, 출시 및 저작권 등록 (2주)**
- **Phase 4: 정식 버전 고도화 및 확장 (지속)**

--- 

## Phase 0: 준비 및 설계 (기간: 2주)

- **상태**: **완료**

## Phase 1: 프로젝트 기반 구축 및 인증 개발 (기간: 2주)

- **목표**: 3개(프론트, 백엔드, 시그널링) 프로젝트의 기본 구조를 설정하고, 시스템의 기반이 되는 사용자 인증 기능을 구현합니다.
- **주요 마일스톤**:
  1.  **[FE/BE/Signal]** **프로젝트 초기 설정**
      - **Frontend**: Next.js 프로젝트 생성 및 폴더 구조 설정.
      - **Backend**: Spring Boot 프로젝트 생성 및 계층형 아키텍처 구조 설정.
      - **Signaling**: Node.js 프로젝트 생성 및 Socket.IO, Redis 연동 설정.
  2.  **[BE]** **사용자 인증 API 구현**
      - Spring Security와 JWT를 이용한 토큰 기반 인증 시스템 구축 (소셜 로그인 포함).
  3.  **[Signal]** **WebSocket 인증 구현**
      - 클라이언트가 보낸 JWT를 검증하여 WebSocket 연결을 인가하는 로직 구현.
  4.  **[FE]** **로그인 UI 및 API/Socket 연동**
      - 로그인 페이지 UI 구현 및 백엔드 로그인 API 연동.
      - 로그인 성공 후 받은 토큰으로 시그널링 서버에 WebSocket 연결 및 인증.

## Phase 2: MVP 핵심 기능 개발 (기간: 5주)

- **목표**: MVP 핵심 기능의 백엔드 API, 시그널링 서버 이벤트, 프론트엔드 UI를 병렬로 개발 완료.
- **주요 마일스톤**:
  1.  **[BE/FE]** **스터디 그룹 CRUD** (2주)
      - **Backend**: 스터디 그룹 생성, 조회, 수정, 삭제 API 구현.
      - **Frontend**: 스터디 탐색, 생성, 상세 페이지 UI 및 API 연동.
  2.  **[BE/FE]** **스터디 멤버 관리** (1주)
      - **Backend**: 스터디 가입/탈퇴, 멤버 관리 API 구현.
      - **Frontend**: 스터디 가입 신청, 멤버 목록 보기 등 관련 UI 구현.
  3.  **[All]** **실시간 채팅 및 활동 기능** (2주)
      - **Backend**: 채팅 메시지 저장을 위한 내부 API 구현, 공지사항/캘린더 CRUD API 및 Redis 이벤트 발행 로직 구현.
      - **Signaling**: `send_message` 이벤트 수신 및 처리, Redis 구독을 통한 실시간 알림 전송 로직 구현.
      - **Frontend**: 채팅 UI, 공지사항, 캘린더 UI 구현 및 API/WebSocket 연동.

## Phase 3: MVP 테스트, 출시 및 저작권 등록 (기간: 2주)

- **목표**: 3개 서비스의 통합 테스트를 통해 안정성을 확보하고, MVP 공식 출시 후 소프트웨어 저작권을 등록합니다.
- **주요 마일스톤**:
  1.  **[All]** **통합 QA 및 버그 수정**
      - 3개 서비스 전체를 연동하여 E2E 테스트 시나리오 실행 및 버그 수정.
  2.  **[DevOps]** **배포 환경 구축**
      - Vercel(프론트엔드) 및 AWS(백엔드, 시그널링) 프로덕션 환경 구축 완료.
      - 3개 프로젝트를 위한 CI/CD 파이프라인 완성.
  3.  **[All]** **공식 출시 및 저작권 등록**
      - 프로덕션 환경에 MVP 공식 배포.
      - 완성된 MVP 코드 및 산출물을 기반으로 **소프트웨어 저작권 등록**을 진행.

## Phase 4: 정식 버전(V1) 고도화 및 확장 (지속)

- **목표**: MVP 피드백을 반영하여 서비스를 고도화하고, 지속적으로 새로운 가치를 창출합니다.
- **주요 마일스톤**:
  - **'그룹 마스터' 시스템 개발**: 예치금 및 벌금 자동 정산, 자동 출석 체크 등 관리 자동화 기능 개발.
  - **데이터 기반 동기 부여 시스템 개발**: 개인/그룹 학습 데이터 대시보드, 게이미피케이션 요소 도입.
  - **All-in-One 경험 강화**: WebRTC 기반 화상 스터디, 칸반 보드 등 협업 기능 추가.
  - **성능 최적화**: 대규모 트래픽에 대비한 캐싱 전략, 쿼리 최적화, 인프라 확장.
