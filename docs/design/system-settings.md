# 시스템 설정 페이지 설계 문서

## 📋 설계 일자
2025-01-21

## 🎯 목적
관리자 및 사용자를 위한 전역 시스템 설정 페이지
- **개인 설정** (`/user/settings`): 개인 프로필, 비밀번호, 알림 등
- **시스템 설정** (`/settings`): 전역 설정, 접근성, 언어, 데이터 관리 등

## 🗂️ 페이지 구조

### 경로
- **URL**: `/settings`
- **레이아웃**: 2단 레이아웃 (좌측 메뉴 + 우측 컨텐츠)

## 🔧 기술 스택

- **프레임워크**: Next.js 14 (App Router)
- **스타일**: CSS Modules
- **상태 관리**: React Hooks + LocalStorage
- **API**: 없음 (클라이언트 측 설정만)

## 💾 데이터 저장

```javascript
// localStorage에 저장
const settings = {
  language: 'ko',
  dateFormat: 'YYYY-MM-DD',
  timeFormat: '24h',
  timezone: 'Asia/Seoul',
  theme: 'dark',
  fontSize: 110,
  animations: true,
  accessibility: {
    keyboardNav: true,
    screenReader: true,
    highContrast: false,
  },
  privacy: {
    analytics: true,
    errorReports: false,
    cookiePolicy: 'essential',
  },
  advanced: {
    devMode: false,
    betaFeatures: false,
  }
};

localStorage.setItem('systemSettings', JSON.stringify(settings));
```

## 📝 참고사항

- `/user/settings`는 개인 설정 (프로필, 비밀번호 등)
- `/settings`는 시스템 설정 (언어, 테마, 접근성 등)
- 두 페이지는 완전히 독립적으로 작동

이제 이 설계를 바탕으로 구현을 시작하겠습니다! 🚀
