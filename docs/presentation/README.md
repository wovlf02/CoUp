### 3. Q&A 가이드 (scripts/02-qa-guide.md)
**내용**:
- 20개 예상 질문 및 답변
- 카테고리별 분류 (기술/기능/PM/비즈니스/기타)
- 답변 전략 (STAR 기법)
- 대응 체크리스트

**카테고리**:
1. **기술적 질문** (6개): WebRTC, 아키텍처, DB, 보안 등
2. **기능 및 UX 질문** (4개): 차별점, 모바일, 접근성 등
3. **프로젝트 관리 질문** (4개): 개발 기간, 어려움, 테스트 등
4. **비즈니스 질문** (4개): 수익 모델, 확장성, 경쟁자 등
5. **기타 질문** (2개): 배운 점, 향후 계획 등

## 🎯 사용 방법

### Step 1: 설계 문서 읽기
```
1. 01-slide-design.md 전체 읽기
2. 각 슬라이드의 레이아웃과 내용 파악
3. 15장의 전체 흐름 이해
```

### Step 2: PPT 제작
```
1. 03-production-guide.md 참고
2. PowerPoint 기본 설정 (크기, 색상, 폰트)
3. 슬라이드 1부터 순차적으로 제작
4. 이미지, 아이콘, 로고 준비
5. 애니메이션 추가
```

### Step 3: 시연 영상 제작
```
1. 03-production-guide.md > 8. 시연 영상 제작 가이드 참고
2. OBS Studio로 화면 녹화 (2분)
3. 영상 편집 (속도 조정, 트랜지션)
4. MP4 포맷으로 저장
5. PPT에 임베드
```

### Step 4: 발표 연습
```
1. scripts/01-main-script.md 대본 숙지
2. 첫 연습: 대본 읽으며 전체 흐름 파악
3. 두 번째 연습: 시간 측정 (10분 이내 확인)
4. 세 번째 연습: 자연스럽게 발표
5. 리허설 3회 이상 권장
```

### Step 5: Q&A 준비
```
1. scripts/02-qa-guide.md 읽기
2. 예상 질문 20개 답변 숙지
3. 특히 기술적 질문 집중 준비
4. 모르는 질문 대응 방법 익히기
```

### Step 6: 최종 점검
```
1. PPT 파일 오타 확인 (3회)
2. 영상 재생 테스트
3. 애니메이션 동작 확인
4. 발표 환경 테스트 (프로젝터, 해상도)
5. 백업 파일 준비 (USB, 클라우드)
```

## 💡 발표 Tips

### 발표 전
1. **리허설 3회 이상** (대본 읽기 → 시간 측정 → 자연스럽게)
2. **장비 테스트** (프로젝터, 영상 재생, 애니메이션)
3. **백업 파일 준비** (USB, 클라우드)
4. **충분한 수면** (7-8시간)

### 발표 중
1. **목소리**: 명확한 발음, 적절한 속도, 강약 조절
2. **제스처**: 슬라이드 가리킬 때 명확하게, 과도한 손동작 자제
3. **청중과 소통**: 아이 컨택, 질문 유도
4. **시간 관리**: 각 슬라이드 시간 엄수, 타이머 사용

### Q&A 대응
1. **질문 경청**: 끝까지 듣고 답변
2. **모르면 솔직히**: "좋은 질문이네요, 향후 학습하겠습니다"
3. **구체적 답변**: 추상적 답변 피하기, 기술명/숫자/예시 제시
4. **긍정적 마무리**: 도전 과제도 해결 방안과 함께 제시
